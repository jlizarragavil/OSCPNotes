Password Cracking Methodology

Overview

A structured approach improves efficiency and reduces mistakes during password cracking sessions.

General workflow

  1. Extract hashes
  2. Identify / format hashes
  3. Estimate cracking time
  4. Prepare / mutate wordlists
  5. Perform the attack
1. Extract Hashes

Sources of hashes

  • Database dumps
  • Configuration files
  • Application backups
  • Memory dumps / system files

In penetration tests, hashes are commonly retrieved from compromised systems or exposed databases.

2. Identify & Format Hashes

Hash identification tools

  • hash-identifier
  • hashid

Example usage

hashid <HASH>

Important notes

  • Some hashes may match multiple algorithms (e.g. MD2, MD4, MD5)
  • Manual verification may be required
  • Ensure hash format matches the cracking tool requirements
3. Estimate Cracking Time

Concept

Cracking time = Keyspace / Hash rate

Purpose

  • Determine feasibility of an attack
  • Avoid wasting time on impractical attempts

Considerations

  • Duration of the penetration test
  • Available hardware resources
  • Possibility of using cloud or upgraded systems
4. Prepare Wordlists

Key strategies

  • Use large common wordlists (e.g. rockyou.txt)
  • Apply rule-based mutations
  • Consider known password policies
  • Leverage leaked password databases when available

Approach

  • Start with baseline dictionary attacks
  • Enhance with rules to match expected password patterns
  • Run multiple wordlists if necessary
5. Attack the Hash

Execution

Once preparation is complete, run the cracking tool with the selected hash type, wordlist, and rules.

Example (Hashcat)

hashcat -m <HASH_TYPE> hashes.txt wordlist.txt -r rules.rule

Best practices

  • Double-check hash type and format
  • Avoid extra spaces or line breaks when copying hashes
  • Verify results using multiple tools if needed
Common Pitfalls
  • Incorrect hash type selection
  • Malformed hashes due to formatting issues
  • Underestimating cracking time
  • Ignoring password policy constraints
  • Using insufficient or unoptimized wordlists
Summary
  • A structured methodology improves cracking efficiency.
  • Accurate hash identification is critical.
  • Feasibility analysis avoids wasted effort.
  • Wordlist preparation and rule-based attacks are essential.
  • Attention to detail prevents common operational errors.
