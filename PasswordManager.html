Password Managers Overview

Concept

Password managers store multiple credentials securely using a single master password.

Examples

  • 1Password
  • KeePass

Security model

  • All stored passwords are encrypted
  • Access requires a master password
  • Often integrated with copy/paste or auto-fill features
Attack Scenario

Goal

Extract and crack a password manager database to retrieve the master password.

Typical steps

  1. Gain access to a system
  2. Locate the password manager database
  3. Transfer the database
  4. Convert it into a crackable hash format
  5. Crack the master password
Locating KeePass Database

File type

KeePass databases typically use the .kdbx extension.

Search using PowerShell

Get-ChildItem -Path C:\ -Include *.kdbx -File -Recurse -ErrorAction SilentlyContinue

Example result

  • C:\Users\jason\Documents\Database.kdbx
File Transfer

Objective

Copy the database file from the target system to the attacker machine for offline analysis.

Formatting the Database

Tool used

  • keepass2john (John the Ripper utility)

Command

keepass2john Database.kdbx > keepass.hash

Output

  • Structured hash string representing the KeePass database

Modification

  • Remove prefix like "Database:"
  • Ensure format matches Hashcat requirements
Identifying Hash Type

Hashcat mode lookup

hashcat --help | grep -i KeePass

Result

KeePass corresponds to mode 13400.

Preparing Wordlists

Common wordlist

  • rockyou.txt

Rule set

  • rockyou-30000.rule

Why rules

  • Mutate base words into realistic passwords
  • Increase success rate against human-created passwords
Cracking the Database

Hashcat command

hashcat -m 13400 keepass.hash /usr/share/wordlists/rockyou.txt -r /usr/share/hashcat/rules/rockyou-30000.rule --force

Key parameters

  • -m 13400 → KeePass mode
  • -r → apply rule file
  • rockyou.txt → base dictionary

Result

Recovered master password: qwertyuiop123!

Post-Cracking Access

What happens next

  • Open KeePass with the cracked master password
  • Access stored credentials
  • Retrieve additional usernames and passwords

Impact

Compromise of a single master password can expose all stored secrets.

Key Takeaways
  • Password managers centralize credentials under a master password.
  • Database files (.kdbx) can be extracted from compromised systems.
  • Tools like keepass2john convert databases into crackable hashes.
  • Hashcat mode 13400 is used for KeePass databases.
  • Rule-based dictionary attacks significantly improve cracking success.
