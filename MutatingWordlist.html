Rule-Based Attacks

Concept

Rule-based attacks mutate existing wordlists by applying transformation rules (e.g. capitalization, appending characters) to generate variations that match password policies.

Why use rules

  • Password policies often require uppercase, numbers, and special characters
  • Raw wordlists (e.g. rockyou.txt) usually do not satisfy these constraints
  • Rules automate modifications instead of manually editing wordlists
Wordlist Inspection

View wordlist entries

head /usr/share/wordlists/rockyou.txt

Example output

  • 123456
  • password
  • iloveyou
  • abc123

Most entries do not satisfy strong password policies.

Rule Functions

Common rule operators

  • ^ → prepend a character
  • $ → append a character
  • c → capitalize first letter

Example: append a number

$1

Create rule file

echo \$1 > demo.rule

Test rules (debug mode)

hashcat -r demo.rule --stdout demo.txt

Example output

  • password → password1
  • iloveyou → iloveyou1
Combining Rules

Single line rules (sequential)

$1 c

Applied in order: append number, then capitalize.

Multiple lines (separate rules)

$1
c

Each rule is applied independently.

Example output

  • Password1
  • Password
  • Iloveyou1
  • Iloveyou
Password Policy Mutation

Example requirements

  • Uppercase letter
  • Number
  • Special character

Rule example

$1 c $!

Variants

$! $1 c

Order of rules affects final password transformation.

Cracking with Rule-Based Attacks

Example MD5 cracking

hashcat -m 0 crackme.txt /usr/share/wordlists/rockyou.txt -r demo3.rule --force

Key points

  • -m 0 → MD5 hash mode
  • -r → apply rule file
  • --force → ignore warnings (e.g. no GPU)

Result example

Recovered password: Computer123!

Predefined Rule Sets

Hashcat rule directory

/usr/share/hashcat/rules/

Examples of rule files

  • best64.rule
  • rockyou-30000.rule
  • dive.rule
  • leetspeak.rule

Usage

hashcat -m 0 hashes.txt wordlist.txt -r /usr/share/hashcat/rules/best64.rule
Key Takeaways
  • Rule-based attacks extend dictionary attacks by mutating passwords.
  • They help match real-world password policies.
  • Rules simulate common human password behaviors.
  • Predefined rule sets are highly effective when no policy information is available.
