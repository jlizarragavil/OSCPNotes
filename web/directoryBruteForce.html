Gobuster

Directory brute force

gobuster dir -u http://<IP>/ \
-w /usr/share/wordlists/dirbuster/directory-list-2.3-medium.txt \
-t 50

Basic fuzzing

gobuster dir -u http://<IP>/ -w <WORDLIST> -t 50

Search for common web extensions

gobuster dir -u http://<IP>/ -w <WORDLIST> \
-x php,txt,html,js,json \
-t 50

Linux / Apache / PHP extensions

gobuster dir -u http://<IP>/ -w <WORDLIST> \
-x php,phtml,phps,php3,php4,php5,inc,txt,html,conf,ini,log \
-t 50

Windows / IIS extensions

gobuster dir -u http://<IP>/ -w <WORDLIST> \
-x asp,aspx,ashx,asmx,config,txt \
-t 50

Search for backup files

gobuster dir -u http://<IP>/ -w <WORDLIST> \
-x bak,old,save,tmp,backup,orig,swp \
-t 50

Search for compressed files

gobuster dir -u http://<IP>/ -w <WORDLIST> \
-x zip,tar,gz,tgz,bz2,7z,rar \
-t 50

Search for database files

gobuster dir -u http://<IP>/ -w <WORDLIST> \
-x sql,db,db3,sqlite,sqlite3,dump \
-t 50

Search for config and secret files

gobuster dir -u http://<IP>/ -w <WORDLIST> \
-x env,yaml,yml,json,xml,cfg,cnf,pem,key \
-t 50

Search for Git and SVN repositories

gobuster dir -u http://<IP>/ \
-w <WORDLIST> \
-x git,svn \
-t 50

Filtering (status codes)

gobuster dir -u http://<IP>/ \
-w <WORDLIST> \
-s 200,204,301,302,307,403 \
-t 50

Exclude responses by length

gobuster dir -u http://<IP>/ \
-w <WORDLIST> \
--exclude-length 1234

Hide negative status codes

gobuster dir -u http://<IP>/ \
-w <WORDLIST> \
-b 404,400

Recursive fuzzing

gobuster dir -u http://<IP>/ \
-w <WORDLIST> \
-r

Fuzzing parameters (GET)

gobuster dir \
-u "http://<IP>/<SUBDIRECTORY>/?FUZZ=1" \
-w <WORDLIST>

Fuzzing auth pages (POST)

gobuster dir \
-u http://<IP>/<SUBDIRECTORY>/ \
-w <WORDLIST> \
-m POST \
--data 'uname=Dummy&passwd=FUZZ&submit=Submit' \
-H 'Content-Type: application/x-www-form-urlencoded' \
-t 50

Subdomain enumeration

gobuster dns -d mydomain.com -w <WORDLIST>

gobuster vhost -u http://mydomain.com \
-w <WORDLIST> \
--append-domain

Virtual hosting

gobuster vhost -u http://mydomain.com \
-w <WORDLIST> \
-H 'Host: FUZZ.mydomain.com'

Virtual hosting with IP

gobuster vhost -u http://<IP> \
-w <WORDLIST> \
-H 'Host: FUZZ.thm'

Useful notes

  • .phps may disclose PHP source code.
  • .bak, .old, .save often expose backups.
  • .swp may contain Vim temporary data.
  • .sql, .sqlite, .db may expose credentials.
  • web.config is valuable on IIS servers.
  • 403 responses are often worth investigating.
  • Exclude wildcard responses with --exclude-length.
Feroxbuster

Full scan example

feroxbuster -u http://192.168.45.242:28080 \
-w /usr/share/wordlists/dirbuster/directory-list-2.3-medium.txt \
-t 100 \
-d 1

ffuf

Basic fuzzing

ffuf -c -u http://<IP>/FUZZ -w <WORDLIST> -t 50

Search for files with extensions

ffuf -c -u http://<IP>/FUZZ -w <WORDLIST> \
-e .php,.txt,.html,.js,.json,.bak,.old

Search for sensitive / backup files

ffuf -c -u http://<IP>/FUZZ -w <WORDLIST> \
-e .bak,.old,.save,.swp,.tmp,.backup,.orig

Search for database files

ffuf -c -u http://<IP>/FUZZ -w <WORDLIST> \
-e .sql,.db,.db3,.sqlite,.sqlite3,.dump

Search for config / secret files

ffuf -c -u http://<IP>/FUZZ -w <WORDLIST> \
-e .env,.conf,.cfg,.ini,.json,.xml,.yaml,.yml,.pem,.key

Search for Windows / IIS files

ffuf -c -u http://<IP>/FUZZ -w <WORDLIST> \
-e .asp,.aspx,.config,.ashx,.asmx,.txt

Filtering by status code

ffuf -c -u http://<IP>/FUZZ -w <WORDLIST> \
-fc 404,400

Filtering by size (very important for wildcard responses)

ffuf -c -u http://<IP>/FUZZ -w <WORDLIST> \
-fs 0

Filtering by words / lines

ffuf -c -u http://<IP>/FUZZ -w <WORDLIST> \
-fw 39 -fl 10

Fuzzing parameters (GET)

ffuf -u 'http://<IP>/<SUBDIRECTORY>/?FUZZ=1' \
-c -w <WORDLIST> \
-fs 39

Fuzzing multiple parameters

ffuf -u 'http://<IP>/page.php?FUZZ1=test&FUZZ2=test' \
-c -w <WORDLIST>

Fuzzing auth pages (POST)

ffuf -u http://<IP>/<SUBDIRECTORY>/ \
-c -w <WORDLIST> \
-X POST \
-d 'uname=Dummy&passwd=FUZZ&submit=Submit' \
-H 'Content-Type: application/x-www-form-urlencoded' \
-fs 1435

Login brute force (username + password)

ffuf -u http://<IP>/login \
-X POST \
-d 'username=FUZZ&password=FUZZ2' \
-w users.txt:FUZZ,pass.txt:FUZZ2 \
-c -fs 1234

Subdomain enumeration

ffuf -u http://FUZZ.mydomain.com \
-c -w <WORDLIST> \
-fs 0

Virtual hosting

ffuf -u http://mydomain.com \
-c -w <WORDLIST> \
-H 'Host: FUZZ.mydomain.com' \
-fs 0

Virtual hosting with IP

ffuf -u http://<IP> \
-H 'Host: FUZZ.thm' \
-w <WORDLIST> \
-fs 0

Recursive fuzzing (manual approach)

# Step 1: discover endpoints
ffuf -u http://<IP>/FUZZ -w <WORDLIST>

# Step 2: fuzz discovered paths
ffuf -u http://<IP>/admin/FUZZ -w <WORDLIST>

Extensions + recursion combined

ffuf -c -u http://<IP>/FUZZ -w <WORDLIST> \
-e .php,.txt,.html,.bak,.old \
-recursion

Useful notes

  • ffuf is extremely fast and flexible for fuzzing.
  • -fs is critical to avoid wildcard false positives.
  • -fw / -fl help when responses are similar in size.
  • .env and config files are high-value targets.
  • .git exposure often leads to full source code leakage.
  • Always adjust filtering when switching targets.
  • Combine wordlists + extensions for maximum coverage.
wfuzz

Basic fuzzing

wfuzz -c -w <WORDLIST> http://<IP>/FUZZ

Search for directories and files

wfuzz -c -w <WORDLIST> \
http://<IP>/FUZZ

Search for files with extensions

wfuzz -c -w <WORDLIST> \
-z list,php-txt-html-js-json-bak-old \
http://<IP>/FUZZ.FUZ2Z

Alternative file extension fuzzing

wfuzz -c -w <WORDLIST> \
-z list,.php,.txt,.html,.bak,.old \
http://<IP>/FUZZ

Filtering (hide 403)

wfuzz -c -w <WORDLIST> \
--hc 403 \
http://<IP>/FUZZ

Filtering multiple response codes

wfuzz -c -w <WORDLIST> \
--hc 403,404 \
http://<IP>/FUZZ

Show only valid responses (whitelist)

wfuzz -c -w <WORDLIST> \
--sc 200,301,302,307,403 \
http://<IP>/FUZZ

Filtering by size

wfuzz -c -w <WORDLIST> \
--hh 0 \
http://<IP>/FUZZ

Fuzzing parameters (GET)

wfuzz -c -w <WORDLIST> \
http://<IP>/<SUBDIRECTORY>/?FUZZ=1

Fuzzing multiple parameters

wfuzz -c -w users.txt \
-w pass.txt \
http://<IP>/login?user=FUZZ&pass=FUZ2Z

POST brute force (login)

wfuzz -c -w <WORDLIST> \
-X POST \
-d 'uname=admin&passwd=FUZZ&submit=Submit' \
--hc 403 \
http://<IP>/<SUBDIRECTORY>/

Login brute force (username + password)

wfuzz -c \
-w users.txt \
-w passwords.txt \
-X POST \
-d 'username=FUZZ&password=FUZ2Z' \
--hc 401,403 \
http://<IP>/login

Subdomain enumeration

wfuzz -c -w <WORDLIST> \
--hc 0 \
http://FUZZ.mydomain.com

Virtual hosting

wfuzz -c -w <WORDLIST> \
-H 'Host: FUZZ.mydomain.com' \
--hc 0 \
http://mydomain.com

Virtual hosting with IP

wfuzz -c -w <WORDLIST> \
-H 'Host: FUZZ.thm' \
http://<IP>

Fuzz headers

wfuzz -c -w <WORDLIST> \
-H 'User-Agent: FUZZ' \
http://<IP>/

Cookie fuzzing

wfuzz -c -w <WORDLIST> \
-b 'session=FUZZ' \
http://<IP>/

Recursive fuzzing (manual)

# Step 1
wfuzz -c -w <WORDLIST> http://<IP>/FUZZ

# Step 2
wfuzz -c -w <WORDLIST> http://<IP>/admin/FUZZ
