Password Cracking Concepts

John the Ripper vs Hashcat

  • John the Ripper (JtR): Primarily CPU-based, also supports GPU.
  • Hashcat: Primarily GPU-based, also supports CPU.
  • Hashcat requires OpenCL or CUDA for GPU acceleration.

Hardware considerations

  • GPUs are significantly faster due to thousands of cores.
  • Some hashing algorithms (e.g. bcrypt) perform better on CPUs due to their design.
Keyspace Calculation

Concept

Keyspace represents the total number of possible password combinations.

Formula idea

Keyspace = (character set size) ^ (password length)

Example character set

abcdefghijklmnopqrstuvwxyzABCDEFGHIJKLMNOPQRSTUVWXYZ0123456789

Count character set size

echo -n "abcdefghijklmnopqrstuvwxyzABCDEFGHIJKLMNOPQRSTUVWXYZ0123456789" | wc -c

Calculate keyspace (5 characters)

python3 -c "print(62**5)"

Result interpretation

  • 62 possible characters per position
  • 5-character password → 62^5 combinations
  • Total: 916,132,832 possibilities
Hashcat Benchmark

Run benchmark (CPU/GPU)

hashcat -b

Example hash modes

  • MD5 (Mode 0)
  • SHA1 (Mode 100)
  • SHA-256 (Mode 1400)

Understanding output

  • Hash rates are measured in MH/s (millions of hashes per second)
  • Higher MH/s = faster cracking capability
CPU vs GPU Performance

Example comparison (SHA-256)

  • CPU: ~134.2 MH/s
  • GPU: ~9276.3 MH/s

Conclusion

GPUs significantly outperform CPUs for most hashing algorithms.

Cracking Time Calculation

Formula

Cracking time = Keyspace / Hash rate

Example (SHA-256, 5-character password)

python3 -c "print(916132832 / 134200000)"   # CPU
python3 -c "print(916132832 / 9276300000)"   # GPU

Interpretation

  • CPU: ~7 seconds
  • GPU: < 1 second
Impact of Password Length

Example calculations

# 8 characters
python3 -c "print(62**8)"

# 10 characters
python3 -c "print(62**10)"

Observations

  • 8-character password: hours to crack (GPU)
  • 10-character password: years to crack (GPU)

Key takeaway

Password length has an exponential impact on security, while increasing character complexity grows keyspace polynomially.
