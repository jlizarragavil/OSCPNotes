OSCP Exam Tips & Low Hanging Fruits (Reddit-Based Cheatsheet)

1. Low Hanging Fruit Mindset

Idea clave: Prioriza siempre lo simple antes que lo complejo.

  • Credenciales por defecto
  • Servicios mal configurados
  • Enumeración incompleta
  • Accesos anónimos

Principio OSCP: “No overthinking, enumerate properly first.”

2. Enumeration First, Always
nmap -sC -sV -p- TARGET_IP

Qué buscar:

  • Todos los puertos (no solo comunes)
  • Versiones de servicios
  • Servicios olvidados (FTP, SMB, RPC, etc.)

Tip Reddit: La mayoría de problemas vienen de mala enumeración.

3. Check Default / Weak Credentials
  • admin:admin
  • admin:password
  • guest:guest
  • service-specific default creds
hydra -l admin -P wordlist.txt TARGET_IP http-post-form "/login:username=^USER^&password=^PASS^:F=invalid"

Qué buscar:

  • Login panels
  • APIs sin auth
  • Credenciales reutilizadas
4. Directory / Content Fuzzing
gobuster dir -u http://TARGET_IP -w /usr/share/seclists/Discovery/Web-Content/common.txt

Qué buscar:

  • Paneles ocultos
  • Backups (.zip, .bak, .old)
  • Admin endpoints
  • APIs documentadas
5. Review Each Service Carefully

No te quedes con el banner.

  • FTP → anonymous login?
  • SMB → shares accessible?
  • SSH → weak creds?
  • HTTP → hidden functionality?
smbclient -L //TARGET_IP -N
ftp TARGET_IP
6. Re-Enumeration After Each Access

Regla clave: cada vez que obtienes acceso nuevo, enumera otra vez.

  • Check privileges
  • Check internal network
  • Look for stored creds
  • Inspect configs and scripts
whoami
whoami /priv
hostname
ipconfig
7. Avoid Rabbit Holes

Qué significa: no perder horas en una sola vía sin progreso.

  • Si no avanzas → cambia de vector
  • Si un exploit no funciona → prueba otro servicio
  • Si no hay resultados → re-enumera

Tip Reddit: el examen recompensa consistencia, no obsesión.

8. Use Multiple Wordlists
gobuster dir -u http://TARGET_IP -w /usr/share/seclists/Discovery/Web-Content/directory-list-2.3-medium.txt
  • No dependas de una sola wordlist
  • Usa SecLists variadas
  • Prueba combinaciones pequeñas + grandes
9. Exploit Selection Strategy
  • Busca exploits públicos (searchsploit)
  • Prioriza exploits simples y fiables
  • Evita exploits innecesariamente complejos
searchsploit service_name version
10. General OSCP Success Tips
  • Enumeración > explotación
  • Siempre revisa lo obvio primero
  • No ignores puertos raros
  • Documenta todo desde el inicio
  • Gestiona bien el tiempo del examen
OSCP Exam Flow (Practical)
  1. Full port scan (TCP + UDP if possible)
  2. Service enumeration
  3. Check default credentials
  4. Fuzz web directories
  5. Search for public exploits
  6. Gain initial access
  7. Re-enumerate
  8. Privilege escalation
  9. Repeat for each machine
