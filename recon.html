Recon & Enumeration Cheatsheet (OSCP Style)

1. Nmap Quick Scan (First Look)
sudo nmap -p- --open --min-rate 5000 -sS -n -Pn target_ip -oN openPorts

What you do:

  • Identify attack surface quickly
  • Find initial entry points (web, SMB, AD, etc.)

OSCP mindset: This defines your first attack vector (NOT random enumeration).

2. Full TCP Scan (All Ports)
nmap -p- -T4 -Pn -n target_ip -oN full_ports.txt
nmap -sCV -p $(grep "^[0-9]" openPorts | awk '{print $1}' | cut -d'/' -f1 | paste -sd,) target_ip -oN services

What you want:

  • Service versions (for exploits)
  • Authentication boundaries (SMB, LDAP, WinRM)
  • High-value targets (AD, DBs, web panels)
3. UDP Scan (If Time Allows)
nmap -sU --top-ports 50 target_ip

Use when: TCP gives nothing interesting or box feels “incomplete”.

10. Port 389 - LDAP (Directory Enumeration)

Objective: Extract directory information (users, structure, configs, creds if exposed).

1. Base discovery

ldapsearch -x -H ldap://target_ip -s base

Goal: identify base naming context

2. Basic enumeration

ldapsearch -x -H ldap://target_ip -b "DC=domain,DC=local"

Look for:

  • Users
  • Groups
  • Machines
  • Descriptions / notes

3. User extraction

ldapsearch -x -H ldap://target_ip \
-b "DC=domain,DC=local" \
"(objectClass=user)"

Use for:

  • Password spraying
  • Authentication attacks

4. Credential hunting

ldapsearch -x -H ldap://target_ip -b "DC=domain,DC=local" | grep -i password

Typical leaks:

  • User descriptions
  • Service account notes
  • Legacy configuration data

5. Authenticated enumeration

ldapsearch -x -H ldap://target_ip \
-D "user@domain.local" \
-w "password" \
-b "DC=domain,DC=local"

What changes:

  • More objects become visible
  • Restricted attributes may unlock

6. High value data (if present)

  • Computer attributes
  • Service accounts
  • Potential LAPS / password fields

7. OSCP mindset

  • LDAP = structured information leak
  • Always pivot to other services after enumeration
  • Rarely the exploit → mostly a discovery tool
11. Port 3389 - RDP
xfreerdp /u:user /p:password /v:target_ip
8. Port 139/445 - SMB (General Enumeration)

Objective: Identify shares, misconfigurations, and possible credential exposure.

1. Initial check

smbclient -L //target_ip -N
smbmap -H target_ip

What you want:

  • Anonymous access
  • Readable shares
  • Writable shares

2. Share exploration

smbclient //target_ip/share -N

Look for:

  • Configs (.conf, .ini)
  • Backups (.bak, .old)
  • Scripts (.ps1, .bat, .sh)
  • Credentials in plain text

3. Recursive listing

smbclient //target_ip/share -N -c "recurse ON; prompt OFF; ls"

Goal: map full share structure

4. Writable shares

If writable:

  • Upload files
  • Test execution paths (web / scripts / services)
  • Drop payloads if execution exists
put file.txt

5. Credential hunting

grep -R "password" .
find . -type f -name "*.config"
find . -type f -name "*.txt"

6. OSCP mindset

  • SMB = file-based attack surface
  • Always check before web brute force
  • Most creds in labs come from files
13. Credential Hunting
grep -R "password" .
find / -name "*.conf" 2>/dev/null

OSCP mindset: creds always exist somewhere → never assume clean environment.
