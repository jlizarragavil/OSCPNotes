Checklist

Linux - After Shell
  • Check current user: whoami
  • Check UID 0: id
  • List running processes: ps auxf
  • Check open ports: ss -tulpn or netstat -tulpn
  • Check kernel version: uname -a
  • Check system info: cat /etc/os-release
  • Check SUID binaries: find / -perm -4000 -type f 2>/dev/null
  • Check capabilities: getcap -r / 2>/dev/null
  • Check writable files/directories: find / -writable -type f 2>/dev/null
  • Check scheduled cron jobs: crontab -l, ls -la /etc/cron*
  • Check Docker/LXD groups: groups + id
  • Check PATH hijack opportunities: echo $PATH + look for writable dirs
  • Check for passwords: grep -i password /etc/passwd /etc/shadow /home/*/.bash_history
  • Look for SSH keys: find /home -name '*.pem' -o -name '*.key' 2>/dev/null
  • Run enumeration scripts (LinPEAS, LinEnum) if possible
Windows - After Shell
  • Check current user: whoami
  • Check groups and privileges: whoami /groups
  • Check if admin: net session (will fail if not admin)
  • List running processes: tasklist /v
  • Check listening ports: netstat -ano
  • Check Windows version: systeminfo
  • Enumerate services: sc query
  • List scheduled tasks: schtasks /query /fo LIST /v
  • List SUID-equivalent binaries (Check for dangerous executables and permissions)
  • Look for interesting files: dir /s /b *.txt, dir /s /b *.config
  • Check mapped drives and shares: net use
  • Run PowerUp, WinPEAS, Sherlock scripts if possible
After Getting Credentials
  • Test credentials on SMB / WinRM / RDP / SSH
  • Check local admin access on other machines
  • Dump hashes if allowed: secretsdump.py or impacket-secretsdump
  • Check SPNs for kerberoasting opportunities
  • Use BloodHound / PowerView / AD modules to enumerate rights and object permissions
  • Check for cached creds / RDP / saved passwords
  • Check for interesting shares: Find-DomainShare, net view
  • Look for service accounts / privileged users / DA membership
  • Check for nested groups and delegation
When Stuck
  • Check for misconfigurations in services (ex: MSSQL, FTP, Redis)
  • Try password spraying / common passwords
  • Check for old backup files, database dumps, config files
  • Enumerate users from logs / files / scripts
  • Re-check permissions on files, shares, cron, tasks
  • Re-run automated enumeration tools (LinPEAS, BloodHound, WinPEAS)
  • Look for hardcoded credentials in scripts / binaries
  • Check scheduled tasks / cron jobs for privilege escalation
  • Scan local services for weak configurations
