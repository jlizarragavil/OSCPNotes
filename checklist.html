Checklist

Linux - After Shell
  • Check current user: whoami
    • Confirms your current identity and context
  • Check UID 0: id
    • Look for UID=0 (root)
    • Check sudo privileges
    • Identify groups that may grant access
  • List running processes: ps auxf
    • Look at the USER column → identify processes running as root
    • Check for binaries/scripts in unusual paths (/tmp, /dev/shm, /home)
    • Identify unknown or custom services
    • If root runs a script → check if you can modify it
  • Check open ports: ss -tulpn or netstat -tulpn
    • Look for services bound to 127.0.0.1 (internal only)
    • Identify databases (3306, 5432), Redis, etc.
    • Map ports to processes (PID → process)
    • Check for services you can interact with locally
  • Check kernel version: uname -a
    • Search for kernel exploits if outdated
  • Check system info: cat /etc/os-release
  • Check SUID binaries: find / -perm -4000 -type f 2>/dev/null
    • Identify binaries that run as root
    • Check unusual/custom binaries
    • Search in GTFOBins for exploitation
  • Check capabilities: getcap -r / 2>/dev/null
    • Look for binaries with elevated capabilities
    • Some allow privilege escalation without SUID
  • Check writable files/directories: find / -writable -type f 2>/dev/null
    • Writable scripts executed by root
    • Config files you can modify
    • Anything in cron jobs or services
  • Check scheduled cron jobs: crontab -l, ls -la /etc/cron*
    • Look for scripts executed as root
    • Check if scripts are writable
    • Check for path abuse in cron jobs
  • Check Docker/LXD groups: groups + id
    • May allow container escape
    • Check membership in docker/lxd groups
  • Check PATH hijack opportunities: echo $PATH
    • Look for writable directories in PATH
    • Check if root executes binaries without full path
  • Check for passwords: grep -i password /etc/passwd /etc/shadow /home/*/.bash_history
    • Look for reused or hardcoded credentials
    • Check config files and history files
  • Look for SSH keys: find /home -name '*.pem' -o -name '*.key' 2>/dev/null
    • Private keys can allow lateral movement
  • Run enumeration scripts (LinPEAS, LinEnum) if possible
    • Automates privilege escalation discovery
Windows - After Shell
  • Check current user: whoami
  • Check groups and privileges: whoami /groups
    • Look for SeImpersonatePrivilege
    • Look for SeDebugPrivilege
    • Identify privilege escalation opportunities
  • Check if admin: net session
    • If it works → you are admin
  • List running processes: tasklist /v
    • Identify SYSTEM processes
    • Look for unknown binaries/services
  • Check listening ports: netstat -ano
    • Identify internal services
    • Map PID → process
  • Check Windows version: systeminfo
    • Useful for vulnerability research
  • Enumerate services: sc query
    • Look for misconfigured services
    • Check service binaries and permissions
  • List scheduled tasks: schtasks /query /fo LIST /v
    • Check tasks running as SYSTEM
    • Verify executable paths are writable
  • List SUID-equivalent binaries
    • Check weak file permissions on executables
  • Look for interesting files: dir /s /b *.txt, dir /s /b *.config
    • Config files often contain credentials
  • Check mapped drives and shares: net use
    • Look for accessible shares and data
  • Run PowerUp, WinPEAS, Sherlock scripts if possible
    • Automates enumeration and privesc discovery
After Getting Credentials
  • Test credentials on SMB / WinRM / RDP / SSH
    • Verify where credentials are valid
  • Check local admin access on other machines
  • Dump hashes if allowed: secretsdump.py or impacket-secretsdump
    • Used for lateral movement
  • Check SPNs for kerberoasting opportunities
    • Target service accounts in AD
  • Use BloodHound / PowerView / AD modules
    • Map relationships and attack paths
  • Check for cached creds / RDP / saved passwords
  • Check shares: net view
    • Look for sensitive files or credentials
  • Look for service accounts / privileged users
  • Check nested groups and delegation
    • Privilege escalation via group nesting
When Stuck
  • Check misconfigurations (MSSQL, FTP, Redis)
    • Default credentials or weak auth
  • Try password spraying / common passwords
  • Look for backups, dumps, config files
    • Often contain credentials
  • Enumerate users from logs/scripts
  • Re-check permissions on files and tasks
  • Re-run enumeration tools (LinPEAS, BloodHound, WinPEAS)
  • Look for hardcoded credentials
  • Check scheduled tasks / cron jobs
  • Scan local services for weak configs
Password Cracking Methodology
  • Extract hashes from the target (databases, dumps, files)
  • Identify hash type using tools like hashid or hash-identifier
  • Format hashes correctly for the cracking tool
  • Estimate cracking feasibility (time vs keyspace vs hash rate)
  • Prepare and customize wordlists (rules, mutations, leak data)
  • Perform the cracking attack using tools like Hashcat or John
  • Validate hash input carefully (avoid extra spaces or newlines)
  • Double-check hash type if ambiguous
  • Iterate with different wordlists and rules if needed
  • Consider hardware or cloud resources for long cracking sessions
