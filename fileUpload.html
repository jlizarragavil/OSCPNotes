Concept

File upload vulnerabilities occur when a web application allows users to upload files without proper validation.

Even if uploaded files are not directly executable, they can still be abused for:

  • Directory Traversal
  • File Overwrite
  • Information disclosure
  • Privilege escalation
Detection

Basic tests

  • Upload a harmless file (e.g. test.txt)
  • Check if the file is stored and accessible
  • Upload the same file twice
  • Observe error messages or overwrite behavior

Intercept upload (Burp)

  • Capture request
  • Check filename parameter
  • Inspect response for hints
Exploitation

Filename manipulation

filename=../../../../../../../test.txt

Goal

  • Write files outside web root
  • Overwrite sensitive files
  • Attempt privilege escalation
Directory Traversal Abuse

Example overwrite target

../../../../../../../root/.ssh/authorized_keys

Attack steps

  1. Create SSH keypair
  2. Copy public key into authorized_keys
  3. Upload file with traversal path in filename
  4. Overwrite target file
  5. Authenticate via SSH using private key

SSH connection example

ssh -p 2222 -i fileup root@target
Privilege Escalation

Common targets

  • /root/.ssh/authorized_keys
  • /home/user/.ssh/authorized_keys
  • Application config files
  • Web server files

Goal

Inject SSH key or modify configuration to gain access as privileged user.

Notes
  • Web servers often run as www-data (Linux) or IIS identities (Windows)
  • Applications may run with elevated privileges if misconfigured
  • Blind overwrites can cause downtime
  • Traversal behavior depends on server-side validation
