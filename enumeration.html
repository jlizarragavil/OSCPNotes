Exercises

Initial Access – Assumed Breach

Since we are starting in an assumed breach scenario and we have credentials for stephanie, we will use those credentials to authenticate to the domain via a Windows 11 machine (CLIENT75). We will use the Remote Desktop Protocol (RDP) with xfreerdp to connect to the client and log in to the domain. We will supply the username with /u, the domain name with /d, and then enter the password, which in this case is LegmanTeamBenzoin!!. 


xfreerdp3 /u:stephanie /d:corp.com /v:192.168.148.75

1. Check User in Domain Admins Group

Start VM Group 2 and log in to CLIENT75 as stephanie. Using PowerView, enumerate the modified corp.com domain and determine which new user is a member of the Domain Admins group.

First, search for the PowerView script on the system:


Get-ChildItem C:\ -Filter PowerView.ps1 -Recurse -ErrorAction SilentlyContinue

Once PowerView is available, enumerate the Domain Admins group:


Get-NetGroup "Domain Admins" | select member

2. Enumerate User Office (Flag)

Continue enumerating the corp.com domain in VM Group 2. Enumerate which Office the user fred is working in to obtain the flag. 


Get-NetUser "fred"

3. Enumerate Server Distinguished Name

Start VM Group 1 and log in to CLIENT75 as stephanie. Repeat the PowerView enumeration steps outlined earlier to enumerate the computer objects in the domain. Identify the DistinguishedName for the WEB04 machine. 


xfreerdp3 /u:stephanie /d:corp.com /v:192.168.124.75
        

Get-ChildItem C:\ -Filter PowerView.ps1 -Recurse -ErrorAction SilentlyContinue
Set-ExecutionPolicy -Scope Process -ExecutionPolicy Bypass
Import-Module .\PowerView.ps1

Enumerate the computer objects in the domain:


Get-NetComputer | select operatingsystem,dnshostname,distinguishedname

Answer: CN=web04,CN=Computers,DC=corp,DC=com

4. Enumerate Server Operating System Version

Continue enumerating the operating systems in VM Group 1. Identify the exact operating system version for FILES04. The answer should be provided in the format xx.x (xxxxx). 


Get-NetComputer | select operatingsystemversion,dnshostname

Answer: 10.0 (20348)

5. Enumerate Operating Systems (Flag)

Start VM Group 2 and log in to CLIENT75 as stephanie. Use PowerView to enumerate the operating systems in the modified corp.com domain and obtain the flag. 


Get-NetComputer | select operatingsystem
        

OS{fdaf473fcc28a3d71438120fc983108f}

6. Enumerate Permissions and Local Admin Access

Start VM Group 2 and log in to CLIENT75 as stephanie. Determine which machines the current user has local administrative privileges on, then log in to one of those machines and obtain the flag from the Administrator Desktop. 


Find-LocalAdminAccess

From the output, we can see that stephanie has administrative access to client74 and web04. We then resolve their IP addresses: 


nslookup client74.corp.com
nslookup web04.corp.com

After connecting via RDP to web04, the flag can be found on the Administrator Desktop: 


type C:\Users\Administrator\Desktop\proof.txt
OS{c63ddf8335a6c2c6a4c116488b682e76}

7. Enumerate Service Principal Names (SPN)

Start VM Group 1 and enumerate the service accounts in the domain. Identify the unique service identifier used to associate a service with Active Directory. 


Get-NetUser -SPN | select samaccountname,serviceprincipalname

Answer: Service Principal Name (SPN)

8. Enumerate SYSVOL Share

Enumerate the accessible domain shares and determine the hostname of the server sharing the SYSVOL folder in the corp.com domain. 


Find-DomainShare

Answer: dc1

9. Enumerate Accessible Shares (Flag)

Start VM Group 2 and use PowerView to locate and enumerate accessible shares in the modified corp.com domain to obtain the flag. 


Find-DomainShare -CheckShareAccess

After identifying accessible shares on FILES04, we enumerate the Important Files share and retrieve the flag: 


ls "\\FILES04.corp.com\Important Files"
type "\\FILES04.corp.com\Important Files\proof.txt"
OS{ffd0817082c4611deab11249dc95b8bc}

Tips

A fast way to identify interesting files across all accessible shares is to recursively search for .txt files: 


Find-DomainShare -CheckShareAccess | ForEach-Object {
  Get-ChildItem "\\$($_.ComputerName)\$($_.Name)" -Recurse -ErrorAction SilentlyContinue -Include *.txt
}
Enumeration from Linux (Kali)

Phase 0 – Initial Scan

Full port scan


nmap -p- --min-rate 5000 -T4 <IP>

Service scan


nmap -sCV -p <ports> <IP>

Important AD ports

  • 88 → Kerberos
  • 389 → LDAP
  • 445 → SMB
  • 5985 → WinRM
  • 3268 → Global Catalog

Phase 1 – No Credentials

SMB Anonymous


smbclient -L //<IP> -N

LDAP Anonymous


ldapsearch -x -H ldap://<IP> -s base
